# [Weekly Recap] 데이터의 흐름으로 증명하는 성장과 무중단 전환의 미학

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

2026년 7월의 두 번째 주를 마무리하며, 지난 한 주간의 개발 기록과 기술적 고민들을 정리해 봅니다. 이번 주는 단순한 데이터 업데이트부터 고위험 운영 환경의 시스템 전환까지, '측정'과 '안정성'이라는 두 가지 키워드가 중심이 된 시간이었습니다.

### 🛠️ 이번 주의 주요 작업: 측정과 안정성의 조화

**1. 데이터 기반의 자기 객관화: 대시보드 자동화**
한 주 내내 `dashboard.json`을 업데이트하며 저의 개발 궤적을 시각화하는 작업에 집중했습니다. 표면적으로는 단순한 `chore` 작업이었지만, 그 본질은 **'측정 가능한 성장'**에 있었습니다. 
- `heritageradio` 프로젝트의 커밋 수가 급증하는 것을 보며 설계 단계에서 구현 단계로의 전이를 확인했고, `mirae` 리포지토리의 ADR 추적 스크립트 작업을 통해 도메인 확장성을 실험했습니다.
- 특히 '주간 리캡'에서 '일일 포스트'로 기록 파이프라인을 변경하며, Web3와 AI라는 격변의 분야에서 더 세밀한 기록 단위가 필요함을 절감했습니다.

**2. 무중단 시스템 전환: Tier3 아토믹 스왑(Atomic Swap)**
가장 긴장감 있었던 작업은 운영 중인 HLS 서빙 경로를 날짜 기반(`d<YYYYMMDD>/`)으로 변경하는 것이었습니다. 1초의 끊김도 허용되지 않는 환경에서 저는 **'Shadow Upload'** 전략을 채택했습니다.
- **S1(재작성) $\rightarrow$ S2(섀도우 업로드) $\rightarrow$ S3(카나리 플립)**로 이어지는 단계적 접근을 통해 운영 영향도를 '0'으로 수렴시켰습니다.
- `.ts` 세그먼트를 먼저 업로드하고 매니페스트 파일은 유지함으로써, 검증이 완료되기 전까지는 사용자에게 노출되지 않는 안전장치를 마련했습니다. 시니어 엔지니어로서 리스크를 어떻게 통제하고 검증 루프를 설계해야 하는지를 다시금 복기한 사례였습니다.

### 💡 기술적 인사이트: 시스템과 예술, 그리고 본질

이번 주에는 기술적인 구현 외에도 윌리엄 서머싯 몸의 **《달과 6펜스》**를 읽으며 엔지니어의 삶에 대해 깊이 고민했습니다. 
사회가 정의하는 안정(6펜스)과 내면의 열망(달) 사이의 갈등은 기술자에게도 동일하게 적용됩니다. 효율적인 아키텍처와 표준이라는 '6펜스'의 세계에 안주하지 않고, 본질적인 혁신과 파괴적 창조라는 '달'을 향해 나아가는 태도가 결국 진정한 기술적 도약을 만든다는 점을 깨달았습니다.

### 🚀 다음 주를 향한 생각

데이터를 수동으로 업데이트하며 느낀 '휴먼 에러'의 가능성을 완전히 제거하고 싶습니다. 다음 주에는 GitHub Actions를 활용해 API 호출부터 JSON 업데이트까지 이어지는 **완전 자동화 CI/CD 파이프라인**을 구축하여, 기록의 비용을 낮추고 인사이트의 밀도를 높이는 데 집중할 계획입니다.

측정되지 않는 것은 개선될 수 없습니다. 다음 주에도 데이터로 증명하고, 안정성으로 신뢰를 주는 개발을 이어가겠습니다.
