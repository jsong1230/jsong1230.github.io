## [Dev Log] 데이터가 말해주는 성장: 대시보드 업데이트와 기록의 힘

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

엔지니어에게 있어 '기록'은 단순히 과거를 남기는 것이 아니라, 현재의 나를 객관적으로 바라보게 하는 가장 강력한 피드백 루프입니다. 저는 개인적으로 제가 어떤 프로젝트에 시간을 쏟고 있는지, 기술적 관심사가 어디로 흐르고 있는지를 시각화하는 대시보드를 운영하고 있습니다.

오늘은 이 대시보드의 기반이 되는 `dashboard.json` 데이터를 업데이트하며, 최근 저의 개발 궤적을 정리하는 시간을 가졌습니다.

### 🛠️ 오늘의 작업: 대시보드 데이터 최신화

오늘의 작업은 단순한 `chore` 커밋처럼 보일 수 있지만, 그 이면에는 최근 제가 집중하고 있는 기술적 탐구들이 고스란히 담겨 있습니다.

```json
// src/data/dashboard.json 변경 사항 중 일부
{
  "name": "heritageradio",
  "last_msg": "chore: gram 로컬 배포 테스트 세팅",
  "branch": "study",
  "commits_7d": 4,
  "commits_30d": 4
},
{
  "name": "openfield",
  "last_msg": "Add Claude Code agent definitions and update .gitignore",
  "branch": "study",
  "commits_7d": 2,
  "commits_30d": 2
}
```

이번 업데이트에서 눈에 띄는 점은 `heritageradio`와 `openfield` 같은 새로운 실험적 저장소들이 추가되었다는 것입니다. 특히 `openfield`에서 **Claude Code agent** 정의를 업데이트한 부분은 제가 최근 AI 에이전트의 자율성과 개발 워크플로우 자동화에 깊은 관심을 두고 있음을 보여줍니다.

### 💡 기술적 결정과 인사이트: '정적 데이터'의 가치

많은 분이 실시간 DB를 사용해 대시보드를 구축하지만, 저는 의도적으로 JSON 파일 기반의 정적 데이터 업데이트 방식을 유지하고 있습니다. 여기에는 몇 가지 전략적 이유가 있습니다.

1. **성능과 비용의 최적화**: 개인 대시보드는 굳이 무거운 런타임 서버가 필요 없습니다. JSON 파일 하나로 정적 호스팅(GitHub Pages 등)을 통해 전 세계 어디서든 빠르게 접근할 수 있습니다.
2. **Git을 통한 이력 관리**: 데이터의 변경 사항이 Git 커밋 로그로 남기 때문에, "내가 언제부터 이 프로젝트에 몰입했는가"를 `git diff`만으로 확인할 수 있습니다.
3. **강제적인 회고**: 자동화된 파이프라인이 데이터를 밀어 넣지만, 최종 커밋을 확인하는 과정에서 자연스럽게 한 주 혹은 하루의 활동을 복기하게 됩니다.

### 🚀 마치며

20년 넘게 엔지니어로 살며 깨달은 점은, 뛰어난 기술 스택보다 더 중요한 것은 **'지속 가능한 학습 습관'**이라는 것입니다. 블록체인, Web3, DID, 그리고 이제는 AI 에이전트까지. 기술의 파도는 빠르게 밀려오지만, 이를 기록하고 시각화하여 내 것으로 만드는 과정이 있다면 어떤 변화 앞에서도 당황하지 않을 수 있습니다.

씨피랩스에서도 이러한 '기록의 문화'와 '데이터 기반의 회고'를 팀 전체로 확산시켜, 함께 성장하는 엔지니어링 문화를 만들어가고자 합니다.

내일은 또 어떤 새로운 도전이 기다리고 있을까요? 기록하는 개발자로 계속 나아가겠습니다.
