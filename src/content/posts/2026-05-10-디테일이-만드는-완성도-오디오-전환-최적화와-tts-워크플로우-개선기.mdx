# [Dev Log] 디테일이 만드는 완성도: 오디오 전환 최적화와 TTS 워크플로우 조정

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 시스템의 거대한 아키텍처를 설계하는 것도 중요하지만, 실제 사용자가 느끼는 '제품의 완성도'는 아주 작은 디테일에서 결정되곤 합니다. 특히 오디오 콘텐츠 서비스에서는 0.1초의 정적이나 부자연스러운 전환이 전체 청취 경험을 해칠 수 있습니다. 오늘은 `heritageradio` 프로젝트에서 진행한 오디오 빌더 최적화 작업과 TTS 엔진 설정 변경에 대해 공유해보고자 합니다.

### 1. '거친 전환'을 잡는 디테일: Acrossfade 최적화

최근 내부 피드백을 통해 특정 구간(1955 DRAMA_THEATER 등)에서 목소리(voice)가 끝나고 배경음(tail)으로 넘어가는 지점이 다소 거칠게 들린다는 점을 발견했습니다. 

기존에는 `voice`에서 `tail`로 넘어갈 때 1.5초의 크로스페이드(Acrossfade)를 적용했으나, 콘텐츠의 성격에 따라 이 시간이 짧아 뚝 끊기는 느낌을 준 것이었습니다. 이를 해결하기 위해 페이드 길이를 3.0초로 상향 조정했습니다.

```python
# 5-10 피드백 반영: voice → tail/music 전환을 1.5s에서 3.0s로 상향
_XFADE_DUR_BY_PAIR = {
    ("signal", "voice"): 0.4,
    ("voice", "tail"): 3.0,  # 1.5 -> 3.0
    ("voice", "music"): 3.0, # 1.5 -> 3.0
    ...
}
```

단순히 숫자 하나를 바꾼 것이지만, 오디오의 흐름이 훨씬 부드러워졌습니다. 기술적 결정의 근거는 '일관성'이었습니다. `voice`에서 `tail`로 갈 때와 `music`으로 갈 때의 대칭을 맞춤으로써 전체적인 청취 톤앤매너를 통일했습니다.

### 2. Trailing Silence 제거: 음악 슬롯의 빈틈 채우기

또 다른 문제는 음악 슬롯 끝부분에 발생하는 원치 않는 정적(Silence)이었습니다. 음악 트랙들의 총 합이 설정된 슬롯 길이보다 짧을 경우, 끝부분이 비어버리는 현상이 발생했습니다.

이를 해결하기 위해 `voice` 슬롯에 적용했던 'tail-music chain' 로직을 음악 슬롯에도 동일하게 확장 적용했습니다. 콘텐츠 길이가 부족하면 자동으로 `tail-music` 체인을 호출해 끝부분을 자연스럽게 채우도록 구현했습니다.

```python
content_sec = sum(_ffprobe_dur(p["path"]) for p in pieces)
target_tail = slot_dur - content_sec
if target_tail >= MIN_TAIL_TARGET and ADMIN_TOKEN:
    # 부족한 시간을 계산해 tail-music chain으로 채움
    chain = await _fetch_admin_tail(...)
```

이 작업을 통해 "음악이 갑자기 끊기고 정적이 흐르는" 현상을 제거하고, 끝까지 밀도 있는 사운드 경험을 제공할 수 있게 되었습니다.

### 3. TTS 엔진 전략의 유연한 변경

마지막으로 `BEAUTIFUL_KOREA` 프로그램의 TTS 엔진을 `qwen3tts`에서 `typecast`로 변경했습니다. 

처음에는 자동 합성이 가능한 엔진을 고려했으나, 콘텐츠의 퀄리티를 극대화하기 위해 외부에서 정교하게 합성 후 업로드하는 '수동 관리 워크플로우'가 더 적합하다는 판단을 내렸습니다. 프론트엔드 코드에서 엔진 타입을 변경함으로써 UI 상의 라벨 색상을 변경하고, 자동 합성 트리거를 비활성화하여 운영 실수 가능성을 차단했습니다.

### 마치며: 엔지니어의 시선과 사용자의 귀

오늘의 작업은 복잡한 알고리즘을 구현한 것은 아니었습니다. 하지만 **"사용자가 어떻게 듣는가"**라는 관점에서 접근했을 때, 코드 한 줄의 수정이 사용자 경험(UX)에 얼마나 큰 영향을 미치는지 다시 한번 느꼈습니다.

씨피랩스에서는 기술적 효율성뿐만 아니라, 실제 제품이 전달하는 '감성적 완성도'까지 챙기는 엔지니어링을 지향합니다. 앞으로도 작은 디테일을 놓치지 않는 집요함으로 더 나은 서비스를 만들어가겠습니다.
