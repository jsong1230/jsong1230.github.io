# [Engineering Log] Divergence는 곧 사고다: DB 동기화 파이프라인의 단일화와 인시던트 해결

안녕하세요, 씨피랩스의 송주한입니다.

엔지니어로서 가장 경계해야 할 것 중 하나가 바로 **'유사한 기능을 하는 두 개의 경로(Divergence)'**가 존재하는 상황입니다. "거의 비슷하니까 괜찮겠지"라고 생각한 지점이 결국 운영 환경에서는 치명적인 'Footgun(자신의 발등을 찍는 실수)'이 되어 돌아오곤 합니다.

오늘 저는 `heritageradio` 프로젝트에서 발생한 DB 동기화 인시던트를 해결하며, 파편화된 배포 스크립트를 하나로 통합하는 작업을 진행했습니다.

### 1. 문제의 발단: 왜 사고가 났는가?

최근 DB 동기화 과정에서 콘텐츠가 프리즈(Freeze)되는 인시던트가 발생했습니다. 원인을 분석해 보니, 자동화된 cron 스크립트와 수동으로 실행하는 `deploy_prod.sh` 내의 `sync_data` 로직이 서로 달랐기 때문입니다.

*   **자동화 스크립트:** allowlist를 통해 필요한 테이블만 덤프하고, 외래키(FK) 제약 조건을 안전하게 처리하는 'A방식'을 사용.
*   **수동 스크립트 (`deploy_prod.sh`):** 전체 덤프를 수행하고, 기본 `run-task` 설정을 그대로 사용해 `--clean` 옵션이 동작함 $\rightarrow$ FK 제약 조건으로 인해 `exit 1` 발생 및 데이터 오염 위험.

결국, 수동 배포를 시도할 때마다 이미 해결했던 `api_keys` 유출 위험이 재발하거나, DB 복구 작업이 실패하는 구조적인 결함이 있었던 것입니다.

### 2. 기술적 결정: "단일 진실 공급원(Single Source of Truth)"

이 문제를 해결하기 위한 가장 근본적인 방법은 **로직의 위임(Delegation)**입니다. `deploy_prod.sh` 안에 인라인으로 구현되어 있던 복잡한 덤프/복구 로직을 모두 제거하고, 검증된 표준 스크립트인 `sync_to_aws_prod.sh`로 호출 경로를 단일화했습니다.

**코드 변경의 핵심 (Diff):**
```bash
# AS-IS: 60줄에 달하는 복잡한 인라인 로직 (pg_dump, s3 cp, ecs run-task...)
- sync_data() {
-   local dump_remote="/tmp/grnal_prod_dump_${TAG_DATE}.dump"
-   ... (중략) ...
-   ssh "$BUILD_HOST" "aws s3 sync ..."
- }

# TO-BE: 검증된 표준 스크립트로 위임
+ sync_data() {
+   local args=""
+   [[ $DRY_RUN -eq 1 ]] && args="--dry-run"
+   log "delegate DB/audio prod sync → sync_to_aws_prod.sh on $BUILD_HOST (allowlist + A방식 FK-safe)"
+   ssh "$BUILD_HOST" "cd ~/heritageradio && AWS_PROFILE=heritageradio bash scripts/sync_to_aws_prod.sh $args"
+ }
```

이렇게 하면 앞으로 어떤 경로로 동기화를 수행하든 동일한 `allowlist`와 `FK-safe` 로직이 적용됩니다. 설정의 drift(편차)를 원천 차단한 것입니다.

### 3. 인시던트를 통해 배운 인사이트

이번 작업 과정에서 RDS의 특이한 동작 두 가지를 추가로 규명했습니다.
1. **PGPASSWORD 덮어쓰기 문제:** 인증 과정에서 환경 변수가 덮어씌워지며 `exit 2`가 발생하는 케이스를 수정했습니다.
2. **권한 및 세션 분리:** `--disable-triggers` 옵션 사용 시 권한 거부 문제가 발생함을 확인하고, `PGOPTIONS replica` 세션을 통해 안전하게 데이터를 복원하는 방식으로 최적화했습니다.

### 마치며

20년 넘게 엔지니어로 일하며 느낀 점은, **최고의 최적화는 기능을 추가하는 것이 아니라 중복을 제거하는 것**이라는 점입니다. 특히 인프라 스크립트에서 "비슷한 로직이 두 군데 있다"는 것은 언제 터질지 모르는 시한폭탄을 안고 있는 것과 같습니다.

오늘의 교훈은 명확합니다. **"Divergence를 제거하라. 그리고 단일한 경로(Canonical path)를 믿어라."**
