## [Insight] 데이터의 흐름으로 읽는 나의 성장 기록: 대시보드 업데이트와 회고

안녕하세요, 씨피랩스에서 CTO로 활동하고 있는 송주한(Jeffrey)입니다.

엔지니어에게 있어 '측정할 수 없는 것은 개선할 수 없다'는 말은 진리입니다. 저는 단순히 코드를 짜는 것에 그치지 않고, 제가 어떤 리듬으로 일하고 어디에 에너지를 쏟고 있는지 시각화하는 것을 좋아합니다. 그래서 개인적으로 운영하는 개발 대시보드(`dashboard.json`)를 통해 저의 커밋 활동과 프로젝트별 기여도를 추적하고 있습니다.

오늘 진행한 작업은 단순한 데이터 업데이트(`chore: Update dashboard data`)였지만, 그 속에 담긴 숫자들을 통해 지난 한 주를 되돌아보는 시간을 가졌습니다.

### 1. 데이터가 말해주는 '몰입의 리듬'

이번 업데이트에서 가장 흥미로운 부분은 프로젝트별 `activity` 배열의 변화입니다. 

```json
"activity": [
  2,
  2,
  2,
  2,
  3,
  3,
  0
]
```

위의 `jsong1230.github.io` 리포지토리의 활동 내역을 보면, 주중 내내 꾸준한 빈도로 기록을 남겼음을 알 수 있습니다. 특히 주말 직전까지 텐션을 유지하며 `commits_30d` 수치가 72에서 73으로 소폭 상승한 점이 눈에 띕니다. 

반면, 다른 프로젝트들의 활동 그래프는 매우 다이내믹합니다. 어떤 날은 0건이었다가, 어떤 날은 52건의 커밋이 쏟아지기도 하죠. 이는 제가 업무를 처리하는 방식이 '완만한 지속'보다는 '강렬한 몰입(Deep Work)'에 가깝다는 것을 보여줍니다. 블록체인이나 Web3 같은 복잡한 아키텍처를 설계할 때는 이러한 집중적인 몰입 시간이 필수적이며, 대시보드의 숫자가 이를 증명해주고 있습니다.

### 2. 기술적 결정: 정적 데이터 기반의 가벼운 추적

많은 분이 이런 대시보드를 위해 무거운 DB나 복잡한 API를 구축하곤 합니다. 하지만 저는 `dashboard.json`이라는 단순한 정적 파일 형식을 유지하는 결정을 내렸습니다.

*   **낮은 오버헤드:** 별도의 서버 없이 GitHub Pages만으로도 충분히 시각화가 가능합니다.
*   **버전 관리:** 데이터의 변화 자체가 Git 커밋 히스토리에 남기 때문에, "언제 나의 생산성이 변했는가"를 `git diff`만으로 추적할 수 있습니다.
*   **단순함의 미학:** 복잡한 시스템일수록 유지보수 비용이 증가합니다. 핵심 가치(나의 활동 추적)에 집중하기 위해 가장 단순한 구조를 선택한 것입니다.

### 3. 마무리하며: 기록이 주는 힘

오늘의 `diff`는 단순한 숫자 변경처럼 보일 수 있습니다. 하지만 20년 넘게 엔지니어로 살아오며 느낀 점은, 이런 사소한 기록들이 모여 결국 '나만의 패턴'을 만든다는 것입니다.

씨피랩스에서도 팀원들에게 항상 강조하는 것이 바로 '가시성(Visibility)'입니다. 내가 무엇을 고민했고, 어떤 시행착오를 겪었는지 기록으로 남길 때 비로소 팀 전체의 기술적 자산이 됩니다.

여러분은 자신의 성장을 어떻게 측정하고 계신가요? 거창한 툴이 아니더라도, 나만의 작은 지표를 만들어 기록해 보시길 권합니다. 그 기록이 훗날 여러분이 어떤 방향으로 나아가야 할지 알려주는 나침반이 될 것입니다.
