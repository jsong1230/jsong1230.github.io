안녕하세요. 시니어 소프트웨어 엔지니어이자 코드 리뷰어입니다.

오늘 제출하신 커밋들은 전반적으로 **"사용자 경험(UX)의 디테일"**과 **"운영상의 실질적인 문제(캐시 이슈)"**를 빠르게 해결하려는 실용적인 접근이 돋보입니다. 특히 관리자 페이지에서 반복적인 클릭을 줄여주는 자동화 로직과 브라우저 캐시 우회 전략이 인상적입니다.

상세 리뷰를 전달드립니다.

---

## ✅ 잘한 점

1. **사용자 중심의 UX 개선 (Lazy Initialization)**
   - `findCurrentWeekIndices` 함수를 도입하여 사용자가 매번 현재 주차를 찾아 클릭해야 했던 번거로움을 제거했습니다. 특히 `useMemo`를 통해 마운트 시점에 한 번만 계산하고 이를 `useState`의 초기값으로 사용하는 패턴은 불필요한 리렌더링을 방지하는 효율적인 선택입니다.

2. **실무적인 캐시 버스팅(Cache-Busting) 적용**
   - `iframe` 내의 정적 파일(`player.html`)이 자주 변경됨에도 브라우저 캐시로 인해 구버전이 노출되는 문제를 `?_=${Date.now()}` 쿼리 스트링으로 명쾌하게 해결했습니다. 이는 인프라 설정(Cache-Control)을 변경하기 어려운 환경에서 프론트엔드 단에서 취할 수 있는 가장 빠르고 확실한 해결책입니다.

3. **명확하고 상세한 커밋 메시지**
   - `[admin] episode-registry — KST today 기준 주차 default 자동 선택`과 같이 어떤 변경이 있었고, 왜(Why) 이 작업이 필요한지(사용자 불편 제거)를 커밋 메시지에 구체적으로 명시하여 리뷰어와 동료들이 맥락을 빠르게 파악할 수 있게 작성되었습니다.

## 🔧 개선할 점

1. **날짜 계산 로직의 하드코딩 및 잠재적 위험**
   - `const kstOffsetMs = 9 * 60 * 60 * 1000;`와 같이 오프셋을 직접 더하는 방식은 간단하지만, `Date` 객체의 기본 동작과 섞일 때 예기치 못한 버그를 유발할 수 있습니다. 
   - **개선 제안:** `Intl.DateTimeFormat`을 사용하거나 `date-fns-tz` 같은 라이브러리를 사용하여 타임존을 명시적으로 처리하는 것이 더 안전합니다. 또한, `MONTHS` 배열의 인덱스에 의존하는 루프보다는 `find` 메서드를 사용하여 가독성을 높일 수 있습니다.
   ```typescript
     // 개선 예시
     const current = MONTHS.find(m => m.month === todayMonth)
       ?.weeks.findIndex(w => w.days.some(d => d.day === todayDay));
   ```

2. **`useState` 초기화 시의 일관성**
   - `LivePlayerPanel`에서 `const [bust] = useState<string>(() => ...)` 패턴을 사용하셨습니다. 값이 변경되지 않는 상수의 성격이라면 굳이 `useState`에 담아 상태로 관리하기보다, 컴포넌트 외부에서 정의하거나 `useMemo`를 사용하는 것이 상태 관리 비용을 줄이는 길입니다. (물론 현재는 마운트 시점의 값을 고정하기 위한 의도로 보이나, `useMemo`가 더 적절한 시맨틱입니다.)

3. **에러 핸들링 및 Fallback 전략**
   - `findCurrentWeekIndices`에서 범위 밖일 때 `(0, 0)`을 반환하고 있습니다. 현재는 5월 1주차가 기본값이지만, 시간이 흘러 `MONTHS` 데이터가 업데이트되었을 때 `(0, 0)`이 항상 유효한 기본값인지 검토가 필요합니다.

## 💡 오늘의 기술 포인트

**1. Cache-Busting via Query String**
브라우저나 CDN은 URL이 동일하면 캐시된 리소스를 반환합니다. 이때 의미 없는 고유 값(Timestamp, UUID 등)을 쿼리 파라미터로 추가하면, 브라우저는 이를 **새로운 요청**으로 인식하여 서버로부터 최신 파일을 강제로 가져오게 됩니다. 정적 HTML 파일의 잦은 업데이트가 필요한 관리자 도구에서 매우 유용한 패턴입니다.

**2. State Lazy Initialization**
`useState(() => initialValue)` 형태의 함수형 초기화는 컴포넌트가 리렌더링될 때마다 초기화 로직이 실행되는 것을 방지합니다. 오늘 코드에서는 `useMemo`로 먼저 계산 후 전달하셨는데, 이 역시 동일한 목적(비싼 계산 비용 절감)을 달성하는 좋은 방법입니다.
