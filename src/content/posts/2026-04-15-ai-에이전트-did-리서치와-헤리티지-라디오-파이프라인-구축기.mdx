# [Dev Log] AI 에이전트와 과거의 목소리: DID 리서치부터 라디오 파이프라인 구축까지

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한입니다.

오늘은 기술적으로 매우 상반된 두 가지 영역—**'미래의 신원 증명(AI Agent DID)'**과 **'과거의 기록 복원(Heritage Radio)'**—에 집중한 하루였습니다. 엔지니어로서 가장 즐거운 순간은 추상적인 리서치 결과가 실제 동작하는 파이프라인으로 구현될 때인데, 오늘이 딱 그런 날이었습니다.

---

### 1. Heritage Radio: 과거를 재생하는 자동화 파이프라인 (v0.2)

최근 진행 중인 '그날' 프로젝트의 v0.2 업데이트를 완료했습니다. 단순히 데이터를 보여주는 것을 넘어, 1953년부터 1994년까지의 시대상을 '라디오'라는 매체로 복원하는 것이 핵심입니다.

**풀고자 했던 문제:**
방대한 뉴스 아카이브와 가요 데이터를 어떻게 하면 '라디오 방송'처럼 자연스럽게 큐레이션하고, 이를 매일 자동으로 생성할 수 있을까?

**기술적 결정과 구현:**
- **Celery + Redis 기반의 데일리 파이프라인:** 매일 새벽 2시, `collect → TTS → Gemma4 검증 → compose`로 이어지는 백그라운드 워커를 구축했습니다. 
- **LLM 기반의 역할 추론(Role Inference):** 단순히 뉴스 순서대로 배치하는 것이 아니라, 토픽 키워드를 분석해 '감성적 정점(emotional_peak)', '경제(economy)', '문화(culture)' 등의 역할을 부여하고 그에 맞는 인트로 멘트를 생성하도록 설계했습니다.

```python
# agents/build_year_packages.py 중 일부
def infer_role(topic: str) -> str:
    t = topic
    if any(k in t for k in ["사망", "서거", "추모", "3주기", "위령", "장례", "순국", "6.25"]):
        return "emotional_peak"
    if any(k in t for k in ["수출", "공장", "산업", "경제", "고속도로", "건설"]):
        return "economy"
    # ... (중략)
```

**인사이트:**
데이터가 부족한 연도(1953~58년 등)를 인접 연도로 보충하는 로직을 구현하며, '완벽한 데이터'보다 '끊기지 않는 사용자 경험'이 더 중요하다는 점을 다시금 느꼈습니다. 또한, Qwen3-TTS를 통해 시대별 아나운서 음성을 입히고 LUFS -14로 음량을 정규화하는 과정에서, 기술적 디테일이 곧 서비스의 '질감'을 결정한다는 것을 확인했습니다.

### 2. AI Agent DID: 자율 에이전트 시대의 '신뢰' 설계

오후에는 씨피랩스의 미래 전략과 연결될 **AI Agent DID(탈중앙 신원증명)** 리서치를 진행했습니다.

**핵심 인사이트:**
2026년 이후 기업 워크플로우의 상당 부분이 AI 에이전트로 대체될 것입니다. 이때 가장 큰 병목은 **"이 에이전트를 믿을 수 있는가?"**라는 신뢰 문제입니다. 저는 이를 W3C DID와 VC(Verifiable Credentials) 표준으로 해결하는 구조를 설계했습니다.

- **구조:** `AI Agent → DID(앵커) → VC(발급처 증명) → 권한 검증`
- **전략:** 씨피랩스가 보유한 370여 개의 DID 특허와 MCP(Model Context Protocol) 서버 구현 능력을 결합한다면, 단순한 챗봇을 넘어 '검증 가능한 신원을 가진 AI 에이전트 생태계'를 선점할 수 있다는 확신이 들었습니다.

### 마치며

오늘의 작업은 **'과거의 데이터를 어떻게 가치 있게 재생할 것인가'**와 **'미래의 AI 신원을 어떻게 정의할 것인가'**라는 두 가지 질문에 답하는 과정이었습니다. 

결국 블록체인, AI, DID라는 서로 다른 기술들도 '신뢰할 수 있는 데이터의 흐름'을 만든다는 점에서 하나의 맥락으로 이어집니다. 내일은 오늘 구축한 데일리 파이프라인이 생성한 첫 번째 '오늘의 라디오'를 직접 들어보며 튜닝할 예정입니다.
