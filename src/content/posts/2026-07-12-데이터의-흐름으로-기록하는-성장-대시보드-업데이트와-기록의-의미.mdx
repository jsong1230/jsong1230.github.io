## [Dev Log] 데이터의 흐름으로 기록하는 나의 성장, 대시보드 업데이트

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

엔지니어에게 있어 '기록'은 단순히 과거를 남기는 것이 아니라, 자신의 성장 궤적을 데이터로 확인하는 과정이라고 생각합니다. 저는 개인적으로 제가 관리하는 여러 프로젝트의 활동량을 시각화하여 보여주는 대시보드를 운영하고 있습니다. 오늘은 이 대시보드의 기반이 되는 `dashboard.json` 데이터를 업데이트하며 느낀 몇 가지 생각들을 공유해볼까 합니다.

### 단순한 데이터 업데이트, 그 이상의 의미

오늘 작업한 내용은 표면적으로는 매우 단순합니다. `chore: Update dashboard data`라는 커밋 메시지처럼, 최근 일주일간의 커밋 횟수와 마지막 커밋 메시지, 활동 로그를 최신화한 작업입니다.

하지만 이 JSON 파일의 변화(diff)를 자세히 들여다보면, 최근 제가 어떤 기술적 관심사에 집중하고 있었는지가 고스란히 드러납니다.

```json
{
  "name": "mirae",
  "last_date": "2026-07-10",
  "last_msg": "feat: SK하이닉스 ADR 첫거래 추적 adr_track.sh — 나스닥(SKHYV/SKHY)",
  "branch": "main",
  "commits_7d": 5,
  "commits_30d": 33
}
```

특히 이번 업데이트에서 눈에 띄는 부분은 `mirae` 리포지토리에 추가된 ADR(미국 주식 예탁 증서) 추적 스크립트 작업입니다. 블록체인과 Web3, AI라는 메인 도메인 외에도, 금융 데이터의 흐름을 자동화하여 추적하는 쉘 스크립트 작업 같은 '작은 자동화'들이 모여 전체적인 엔지니어링 역량을 완성한다고 믿습니다.

### 기술적 결정: 정적 데이터 기반의 대시보드

많은 분이 실시간 API 연동을 통한 대시보드를 선호하시겠지만, 저는 의도적으로 `dashboard.json`이라는 정적 파일을 통해 데이터를 관리하고 있습니다. 여기에는 몇 가지 실무적인 이유가 있습니다.

1. **성능과 비용의 최적화**: 매번 GitHub API를 호출하는 대신, 빌드 타임이나 정기적인 스케줄러를 통해 데이터를 생성함으로써 페이지 로딩 속도를 극대화하고 API Rate Limit 문제를 원천적으로 차단합니다.
2. **스냅샷으로서의 가치**: Git 커밋 히스토리에 데이터가 남기 때문에, 특정 시점에 내가 얼마나 몰입했는지를 '버전 관리' 관점에서 되돌아볼 수 있습니다.
3. **단순함의 미학**: 복잡한 백엔드 인프라 없이도 GitHub Pages와 같은 정적 호스팅만으로 충분한 가치를 제공할 수 있습니다.

### 인사이트: 측정할 수 없다면 개선할 수 없다

피터 드러커의 말처럼, 측정하지 않는 것은 개선할 수 없습니다. 이번 업데이트를 통해 `jsong1230.github.io` 리포지토리의 활동량이 꾸준히 유지되고 있는 것을 확인했습니다. 특히 `Auto-generate weekly book post`와 같은 자동화 파이프라인이 잘 작동하고 있다는 점이 뿌듯하더군요.

씨피랩스에서 수많은 복잡한 아키텍처를 설계하고 DID나 AI 같은 최첨단 기술을 다루고 있지만, 결국 가장 기본이 되는 것은 **'꾸준한 기록'과 '작은 자동화의 반복'**이라는 점을 다시 한번 깨닫습니다.

여러분은 자신의 성장을 어떻게 측정하고 계신가요? 거창한 툴이 아니더라도, 나만의 '데이터 로그'를 만들어보시는 것을 추천드립니다.
