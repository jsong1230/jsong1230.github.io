# [Dev Log] 무음의 공포를 잡다: HLS 빌더의 엣지 케이스 해결과 폴백 전략

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 가장 당혹스러운 순간 중 하나는 "로컬과 스테이징에서는 완벽했는데, 운영 환경의 특정 조건에서만 발생하는 버그"를 마주했을 때입니다. 특히 오디오 스트리밍 서비스에서 '무음 영역'이 발생한다는 것은 사용자 경험 측면에서 치명적인 사고와 같습니다.

오늘은 주말 사이 발생한 `heritageradio` 서비스의 무음 영역 사고를 분석하고, 이를 해결하며 얻은 기술적 인사이트를 공유하려 합니다.

### 1. 문제의 발견: 왜 12분 동안 정적이 흘렀을까?

사고의 핵심은 **'검증 로직의 경직성'**과 **'환경 간 설정 불일치'**라는 두 가지 축에 있었습니다.

#### 첫 번째 원인: 너무 타이트한 Query Validation
주말용 `voice_50` 체인(50분 슬롯)에서 특정 프로그램(DRAMA_THEATER)의 실제 음성 길이가 약 38분 정도로 짧았습니다. 이 경우, 슬롯을 채우기 위해 필요한 `target_sec`(꼬리 음악 길이)가 700초를 넘어갔습니다.

하지만 API의 입력 검증 로직은 다음과 같이 설정되어 있었습니다.
```python
# 기존 코드
target_sec: int = Query(..., ge=5, le=600, description="정확 fit 목표 길이 (초)")
```
`le=600`(10분)이라는 제한 때문에 700초 이상의 요청이 들어오자 서버는 `422 Unprocessable Entity` 에러를 뱉었고, 결과적으로 꼬리 음악이 하나도 추가되지 않아 약 12분간의 무음 구간이 발생한 것입니다.

#### 두 번째 원인: 운영 환경의 인증 토큰 부재
최근 관리자용 `tail-music` API를 Pixabay 자산 기반으로 업데이트했습니다. 하지만 운영(prod) 환경에는 `ADMIN_TOKEN`이 설정되지 않았고, 이로 인해 API 호출 시 `401 Unauthorized`가 발생하며 폴백(fallback) 없이 프로세스가 종료되었습니다.

### 2. 기술적 결정: 유연한 확장과 안전한 폴백(Fallback)

단순히 숫자를 키우는 것을 넘어, 시스템의 회복 탄력성(Resilience)을 높이는 방향으로 수정했습니다.

**첫째, 검증 범위의 현실화**
`target_sec`의 상한선을 `600`에서 `3000`으로 확장했습니다. 50분 슬롯 전체를 음악으로 채워야 하는 최악의 상황까지 수용할 수 있도록 하여, 어떤 프로그램에서도 밸리데이션 에러로 인해 무음이 발생하는 일을 방지했습니다.

**둘째, 계층적 폴백 전략 도입**
운영 환경과 개발 환경의 자산 동기화 속도가 다를 수 있음을 인정하고, `hls_builder.py`에 다음과 같은 로직을 구현했습니다.
- **Step 1**: `ADMIN_TOKEN`이 존재하고 API 호출이 성공하면 최신 `admin tail-music`(Pixabay) 사용.
- **Step 2**: 토큰이 없거나 API 에러(401 등) 발생 시, 즉시 기존의 `legacy Suno music_pool`로 폴백.

이렇게 함으로써 신규 기능을 배포하더라도 시스템이 완전히 멈추지 않고 '최소한의 정상 동작'을 보장하도록 설계했습니다.

### 3. 마치며: 엔지니어로서의 인사이트

이번 작업을 통해 다시 한번 깨달은 점은 **"제약 조건(Constraint)은 반드시 실제 데이터의 분포를 반영해야 한다"**는 것입니다. 개발자의 직관으로 설정한 `le=600`이라는 숫자가 실제 운영 환경의 엣지 케이스에서는 시스템을 마비시키는 스위치가 될 수 있습니다.

또한, 마이크로서비스나 복잡한 파이프라인을 설계할 때 **'우아한 성능 저하(Graceful Degradation)'** 전략이 얼마나 중요한지 체감했습니다. 최신 기능이 작동하지 않을 때 단순히 에러를 내는 것이 아니라, 조금 낡았더라도 안정적인 기존 방식으로 돌아가는 폴백 구조가 서비스의 가용성을 결정짓습니다.

오늘의 수정으로 이제 주말의 정적은 사라지고, 다시 풍성한 음악이 흐르는 라디오가 되었습니다. 내일은 더 견고한 코드로 뵙겠습니다.
