# [Dev Log] 디테일의 한 끗 차이: UI 정교화와 사용자 경험의 복원

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 시스템의 아키텍처를 설계하고 복잡한 비즈니스 로직을 푸는 것도 즐거운 일이지만, 때로는 사용자가 직접 맞닿는 '마지막 1px'을 조정하는 작업이 가장 까다로우면서도 보람찰 때가 있습니다. 오늘은 진행 중인 '헤리티지 라디오' 프로젝트의 모바일 인터페이스를 정교하게 다듬는 작업을 진행했습니다.

단순한 UI 수정처럼 보일 수 있지만, 그 과정에서 고민했던 기술적 결정과 인사이트를 공유해 봅니다.

### 1. 명시적 간격 확보: `marginTop` 대신 `Spacer View`를 선택한 이유

안드로이드와 iOS, 그리고 웹 환경을 동시에 대응하는 크로스 플랫폼 개발을 하다 보면, 동일한 스타일 값이 기기마다 다르게 렌더링되는 상황을 자주 마주합니다. 이번에도 카드 제목과 토글 버튼 사이의 `marginTop: 28` 값이 안드로이드 일부 환경에서 시각적으로 충분히 반영되지 않는 문제가 있었습니다.

단순히 수치를 높이는 방식은 특정 기기에서 너무 넓은 간격을 만들어낼 위험이 있습니다. 그래서 저는 `marginTop`을 제거하고, 높이가 고정된 빈 `View`를 삽입하는 방식을 선택했습니다.

```tsx
// 변경 전
style={{ marginTop: 28, ... }}

// 변경 후
<View style={{ height: 22 }} />
```

이런 **'Spacer View'** 방식은 레이아웃 엔진이 계산해야 할 상대적 마진 값에 의존하는 대신, 명시적인 픽셀 영역을 차지하게 함으로써 플랫폼 간 시각적 일관성을 훨씬 더 강력하게 보장합니다. 때로는 가장 단순한 방법이 가장 확실한 해결책이 됩니다.

### 2. 사용자 피드백의 복원: 연도 Dot Indicator

이전 작업에서 공간 확보를 위해 잠시 제거했던 '연도 인디케이터(1955~1995)'를 다시 복원했습니다. 조그셔틀을 돌려 연도를 변경할 때, 현재 어떤 시점에 와 있는지 시각적으로 즉각 알 수 있는 피드백이 부족하다는 판단 때문이었습니다.

단순히 텍스트만 나열하는 것이 아니라, 선택된 연도에는 **주황색 큰 점(Orange Dot)**과 **Bold 폰트**를 적용하고, 나머지는 **은은한 황동색(Brass Faint)**으로 처리하여 계층 구조를 명확히 했습니다.

```tsx
{MVP_YEARS.map((y, i) => {
  const on = i === idx;
  return (
    <View key={y} style={{ alignItems: "center", gap: 3 }}>
      <View style={{ 
        width: on ? 10 : 7, 
        height: on ? 10 : 7, 
        backgroundColor: on ? colors.orange : colors.brassFaint 
      }} />
      <Text style={{ 
        color: on ? colors.brass : colors.brassMuted, 
        fontFamily: on ? "NotoSansKR_900Black" : "NotoSansKR_400Regular" 
      }}>
        {y}
      </Text>
    </View>
  );
})}
```

### 3. 마치며: 엔지니어의 시선으로 보는 '디테일'

오늘의 작업은 거대한 기능 구현은 아니었지만, **'사용자가 느끼는 완성도'**는 결국 이런 작은 디테일의 합에서 결정된다는 것을 다시금 느꼈습니다. 

특히 `radio-bundle`을 재생성하며 번들 파일의 해시값이 변경되고 HTML에 반영되는 과정을 반복하며, 최적화된 정적 자산 배포의 중요성도 함께 점검할 수 있었습니다. 

기술적 깊이는 복잡한 알고리즘뿐만 아니라, 사용자가 느끼는 불편함을 예민하게 포착하고 이를 코드로 정교하게 해결하는 과정에서도 증명된다고 믿습니다. 씨피랩스에서도 이러한 디테일에 대한 집요함을 바탕으로 더 나은 제품을 만들어 가겠습니다.
