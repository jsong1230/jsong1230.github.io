# [Dev Log] 데이터의 흐름으로 기록하는 나의 성장, 대시보드 업데이트

안녕하세요, 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '가시화(Visualization)'입니다. 복잡한 시스템의 상태를 한눈에 파악하는 모니터링 툴이 중요하듯, 개인의 성장과 작업 흐름 역시 데이터로 기록하고 시각화했을 때 비로소 객관적인 회고가 가능하기 때문입니다.

오늘은 제 개인 대시보드의 기반이 되는 `dashboard.json` 데이터를 업데이트했습니다. 단순한 데이터 수정처럼 보일 수 있지만, 이 과정에서 제가 추구하는 '엔지니어링 라이프사이클'에 대한 몇 가지 생각을 정리해 보았습니다.

### 🛠 오늘의 작업: 대시보드 데이터 갱신

이번 작업의 핵심은 최근 7일 및 30일간의 커밋 활동량과 레포지토리별 최신 상태를 동기화하는 것이었습니다. 

```json
// 변경 사항 중 일부
"generated_at": "2026-07-17T23:00:01.997Z",
"last_date": "2026-07-17",
"last_msg": "Auto-generate daily posts",
```

특히 흥미로운 부분은 `activity` 배열의 변화입니다. 날짜별 커밋 횟수를 나타내는 이 배열은 단순한 숫자의 나열이 아니라, 제가 특정 시점에 어떤 기술적 고민에 몰입했는지를 보여주는 '열정의 궤적'과 같습니다. 이번 업데이트에서는 `heritageradio` 레포지토리의 활동 데이터가 조정되었고, `claude-agent`와 같은 AI 관련 실험적 프로젝트들의 활동 수치가 반영되었습니다.

### 💡 기술적 결정과 인사이트

저는 씨피랩스에서 CTO로서 수많은 프로젝트를 리딩하며, 자동화되지 않은 반복 작업이 엔지니어의 창의성을 얼마나 갉아먹는지 경험했습니다. 그래서 제 개인 대시보드 역시 최대한 자동화된 파이프라인을 통해 데이터를 생성하도록 설계했습니다.

이번 `chore: Update dashboard data` 커밋에서 배운 점은 다음과 같습니다.

1. **데이터의 정합성과 시점의 중요성**: `generated_at` 필드를 통해 데이터가 생성된 정확한 시점을 기록하는 것은 매우 중요합니다. 분산 시스템에서 타임스탬프가 결정적인 역할을 하듯, 개인 기록에서도 '언제'의 데이터인가가 분석의 기준점이 됩니다.
2. **활동량의 질적 해석**: `commits_7d`나 `commits_30d` 같은 정량적 지표는 때로 착시를 일으킵니다. 단 한 줄의 코드로 수천 명의 사용자 경험을 개선하는 '임팩트 있는 커밋'과 단순한 오타 수정 커밋은 가치가 다르기 때문입니다. 따라서 저는 수치뿐만 아니라 `last_msg`를 함께 기록하여 작업의 맥락을 보존하려 노력합니다.

### 🚀 마치며

Web3, DID, AI와 같이 빠르게 변하는 기술 생태계에서 살아남는 유일한 방법은 '꾸준한 기록'과 '지속적인 학습'이라고 믿습니다. 씨피랩스에서의 실무 경험과 개인적인 연구 활동이 이 대시보드라는 작은 데이터셋에 쌓여, 훗날 더 큰 인사이트로 돌아오길 기대합니다.

단순한 JSON 파일의 업데이트였지만, 다시 한번 저의 최근 행보를 돌아볼 수 있었던 유익한 시간이었습니다. 내일은 또 어떤 도전적인 문제들을 풀게 될지 기대됩니다.
