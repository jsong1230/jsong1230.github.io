# [Dev Log] 데이터의 흐름으로 기록하는 나의 성장, 대시보드 업데이트

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

개발자에게 있어 '기록'은 단순히 과거를 남기는 것이 아니라, 내가 어디에 시간을 쏟았고 어떤 방향으로 성장하고 있는지를 확인하는 이정표와 같습니다. 저는 개인적으로 저의 개발 활동을 시각화하여 보여주는 대시보드를 운영하고 있는데, 오늘 그 데이터를 업데이트하며 몇 가지 생각할 점들을 정리해 보았습니다.

### 🛠 오늘의 작업: 대시보드 데이터 최신화

오늘 진행한 작업은 간단해 보이지만 의미 있는 `chore: Update dashboard data` 작업이었습니다. `dashboard.json` 파일을 업데이트하여 최근 7일 및 30일간의 커밋 활동, 그리고 현재 집중하고 있는 리포지토리들의 상태를 반영했습니다.

이번 업데이트에서 흥미로운 점은 제가 최근 집중하고 있는 프로젝트들의 분포입니다. diff 내용을 보면 다음과 같은 변경 사항이 눈에 띕니다.

```json
+ {
+   "name": "company-page",
+   "last_date": "2026-06-04",
+   "last_msg": "운영(cplabs.io) 패리티 보완: 푸터 법적표기·K-BTF 메뉴·web2x 비",
+   "branch": "dev",
+   "activity": [0, 0, 0, 0, 0, 8, 0],
+   "commits_7d": 8,
+   "commits_30d": 8
+ }
```

특히 **씨피랩스(cplabs.io)**의 컴퍼니 페이지 운영 환경 패리티를 보완하는 작업에 집중했던 흔적이 보입니다. 법적 표기나 메뉴 구성 같은 디테일한 부분은 기술적인 챌린지는 적을지 몰라도, 사용자에게 신뢰를 주는 서비스의 완성도를 결정짓는 매우 중요한 작업입니다.

### 💡 기술적 결정과 인사이트

단순한 JSON 업데이트임에도 제가 이 방식을 고수하는 이유는 **'데이터 기반의 자기 객관화'** 때문입니다.

1. **활동의 시각화 (Activity Array):** `activity: [0, 0, 0, 0, 0, 8, 0]`와 같은 배열 형태의 데이터는 단순 합계보다 훨씬 많은 정보를 줍니다. 특정 요일에 작업이 몰렸는지, 혹은 꾸준한 리듬을 유지하고 있는지를 한눈에 파악할 수 있게 합니다.
2. **컨텍스트 유지:** `last_msg`를 함께 기록함으로써, 시간이 흐른 뒤 대시보드를 보았을 때 "내가 이때 왜 이 프로젝트에 시간을 쏟았지?"라는 질문에 즉각적으로 답할 수 있습니다.

### 🚀 마치며

20년 넘게 엔지니어로 일하며 느낀 점은, 최고의 퍼포먼스는 '내가 무엇을 하고 있는지 정확히 인지하는 것'에서 시작된다는 것입니다. 블록체인, Web3, AI와 같이 빠르게 변하는 기술 스택 속에서 중심을 잡기 위해서는 이러한 작은 기록들이 모여 큰 데이터가 되고, 그것이 곧 나의 커리어가 됩니다.

오늘의 업데이트를 통해 다시 한번 제가 집중해야 할 우선순위를 점검할 수 있었습니다. 내일은 또 어떤 코드와 인사이트로 이 대시보드를 채우게 될지 기대됩니다.
