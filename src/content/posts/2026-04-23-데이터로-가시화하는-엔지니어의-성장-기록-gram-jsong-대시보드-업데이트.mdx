# [DevLog] 데이터의 흐름으로 기록하는 나의 성장 궤적: 대시보드 업데이트

안녕하세요, 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '가시화(Visualization)'입니다. 복잡한 블록체인 아키텍처나 Web3의 DID 시스템을 설계할 때도 마찬가지지만, 개인의 성장과 작업 흐름 역시 데이터로 가시화했을 때 비로소 객관적인 회고가 가능하기 때문입니다.

오늘은 제 개인 프로젝트인 `gram-jsong` 대시보드의 데이터를 업데이트하는 작업을 진행했습니다. 단순한 `chore` 작업처럼 보일 수 있지만, 이 데이터들이 담고 있는 의미를 살펴보면 최근 저의 기술적 관심사와 몰입 지점이 어디에 있는지 명확히 드러납니다.

### 데이터가 말해주는 최근의 몰입

이번 커밋의 `diff`를 보면 여러 저장소의 활동 내역이 업데이트되었습니다. 특히 흥미로운 부분은 `heritageradio` 프로젝트의 변화입니다.

```diff
- "last_msg": "mobile: radio-bundle 재생성 (spacer + 연도 dot 복원)",
+ "last_msg": "랜딩 PREVIEW — Live 상태로 전환 (방송 중)",
```

단순한 UI 수정이나 버그 픽스 단계를 넘어, 이제는 실제 'Live' 상태로 전환하며 서비스의 완성도를 높이는 단계에 진입했음을 알 수 있습니다. 또한 `commits_7d` 수치가 293에서 317로 상승한 것을 보며, 최근 이 프로젝트에 쏟은 에너지가 상당했음을 체감합니다.

또 다른 눈에 띄는 변화는 `claude-agent` 저장소의 활동입니다. 
`"4/22: heritageradio dev branch snapshot + AWS deployment staging notes"`라는 메시지에서 알 수 있듯, 최근 저는 AI 에이전트를 활용한 자동화된 배포 전략과 스냅샷 관리 체계를 구축하는 데 집중하고 있습니다. AI가 단순히 코드를 짜주는 것을 넘어, 인프라의 상태를 기록하고 배포 노트를 관리하는 '오퍼레이션 파트너'로서의 가능성을 실험하고 있는 셈입니다.

### 기술적 결정과 인사이트: 왜 자동화된 대시보드인가?

저는 씨피랩스에서 CTO로 활동하며 수많은 팀의 생산성을 관리해 왔습니다. 그 과정에서 깨달은 점은, 개발자에게 가장 강력한 동기부여는 '내가 얼마나 전진하고 있는가'를 스스로 확인하는 것이라는 점입니다.

그래서 저는 제 개인의 작업 로그를 `dashboard.json`이라는 정형 데이터로 관리하고, 이를 통해 주간/월간 활동량을 트래킹합니다. 

1. **정량적 지표의 힘**: `commits_7d`, `commits_30d` 같은 수치는 때로는 단순해 보이지만, 슬럼프가 왔을 때나 방향성을 잃었을 때 제가 다시 몰입할 수 있는 지표가 됩니다.
2. **컨텍스트의 보존**: `last_msg`를 기록함으로써, 시간이 흐른 뒤 특정 시점에 내가 어떤 고민을 하며 코드를 수정했는지 빠르게 리마인드할 수 있습니다.

### 마치며

오늘의 작업은 단순한 JSON 파일 업데이트였지만, 저에게는 지난 며칠간의 치열한 고민과 실행을 정리하는 '마침표'와 같은 시간이었습니다. 

기술의 트렌드는 빠르게 변합니다. Web3에서 AI로, 다시 DID와 결합된 새로운 형태로 진화하고 있죠. 하지만 그 변화의 파도 속에서 중심을 잡게 해주는 것은 결국 '꾸준한 기록'과 '데이터에 기반한 회고'라고 믿습니다.

내일은 또 어떤 데이터가 제 대시보드를 채우게 될지 기대하며, 오늘의 로그를 마칩니다.
