## [Dev Log] 데이터 자동화가 주는 통찰: 나의 '디지털 대시보드' 업데이트 기록

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한입니다.

엔지니어로서 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 **'가시성(Visibility)'**입니다. 시스템의 상태를 모니터링하는 것뿐만 아니라, 나의 생산성과 학습 궤적을 데이터로 확인하는 과정은 성장을 위한 필수적인 피드백 루프가 됩니다.

오늘은 제가 개인적으로 운영 중인 개발 대시보드의 데이터를 업데이트하며 느낀 몇 가지 인사이트를 공유해보려 합니다.

### 🛠️ 오늘의 작업: 대시보드 데이터 갱신 (`chore: Update dashboard data`)

오늘 진행한 작업은 단순해 보이지만, 저의 최근 관심사와 활동량을 한눈에 보여주는 `dashboard.json` 파일을 업데이트하는 것이었습니다. 

```json
// src/data/dashboard.json 변경 사항 중 일부
- "generated_at": "2026-07-20T23:00:01.363Z",
+ "generated_at": "2026-07-21T23:00:01.934Z",
```

단순한 날짜 변경 같지만, 이 데이터의 이면에는 제가 관리하는 여러 리포지토리의 활동 로그가 담겨 있습니다. 이번 업데이트에서 특히 흥미로운 지점은 **'활동의 전이'**입니다.

### 🔍 기술적 결정과 관찰 포인트

**1. 자동화된 로그의 가치**
이번 커밋에서 `jsong1230.github.io` 리포지토리의 메시지를 보면 `"Auto-generate daily posts"`라는 문구가 보입니다. 저는 매일의 기록을 수동으로 남기기보다, 특정 트리거에 의해 자동으로 생성되는 시스템을 구축해두었습니다. 이는 기록에 드는 '마찰력'을 줄여, 지속 가능한 기록 습관을 만들기 위한 전략적 선택입니다.

**2. 관심사의 이동과 리소스 배분**
`diff`를 통해 최근 7일 및 30일간의 커밋 수(`commits_7d`, `commits_30d`) 변화를 살펴보면, 제가 현재 어디에 에너지를 쏟고 있는지 명확히 드러납니다. 
- `claude-agent` 리포지토리의 활동이 증가한 것을 볼 수 있는데, 이는 최근 AI 에이전트의 효율적인 프롬프트 엔지니어링과 AWS 비용 최적화 분석(Compute SP discount rate breakdown)에 집중하고 있음을 보여줍니다.
- 반면, 씨피랩스의 홈페이지 관련 작업은 안정기에 접어들어 활동량이 조절되고 있는 모습입니다.

**3. 개인적 헬스케어 데이터의 통합**
흥미로운 점은 개발 리포지토리뿐만 아니라 `health`라는 이름의 리포지토리를 통해 개인적인 건강 기록까지 관리하고 있다는 점입니다. 
`"last_msg": "포진(HSV) 재발 기록 추가..."`와 같은 로그는 단순한 개발 기록을 넘어, 내 삶의 모든 변수를 데이터화하여 관리하려는 저의 '엔지니어적 접근 방식'이 반영된 결과입니다.

### 💡 마치며: 데이터는 거짓말을 하지 않는다

많은 이들이 '코드 작성'만이 개발자의 일이라고 생각하지만, 사실 더 중요한 것은 **'어떤 방향으로 나아가고 있는가를 측정하는 것'**입니다. 

오늘의 단순한 JSON 업데이트 작업은 저에게 다음과 같은 질문을 던졌습니다. *"나는 계획했던 학습 경로를 제대로 걷고 있는가? 특정 분야에 과하게 매몰되어 균형을 잃지는 않았는가?"*

Web3, DID, 그리고 AI라는 거대한 기술적 파도 속에서 길을 잃지 않는 방법은 결국 나만의 정교한 '대시보드'를 갖는 것입니다. 여러분도 자신의 성장 궤적을 데이터로 기록해보시는 건 어떨까요?

**Jeffrey Joo-Han Song**
