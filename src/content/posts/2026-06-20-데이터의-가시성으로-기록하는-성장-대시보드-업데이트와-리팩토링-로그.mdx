## [Dev Log] 데이터의 흐름이 보여주는 성장의 궤적: 대시보드 업데이트와 리팩토링의 기록

안녕하세요, 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '가시성(Visibility)'입니다. 시스템의 상태뿐만 아니라, 내가 어떤 방향으로 시간을 쓰고 있는지, 기술적 부채는 어떻게 해결되고 있는지를 데이터로 확인하는 과정은 단순한 기록 이상의 의미가 있습니다.

오늘은 개인 대시보드의 데이터를 업데이트하며, 최근 진행 중인 씨피랩스의 프로젝트들과 개인적인 연구 과제들의 진척도를 정리했습니다. 단순한 `json` 파일 업데이트처럼 보이지만, 그 변경 사항(diff) 속에는 최근 제가 집중하고 있는 기술적 고민들이 고스란히 담겨 있습니다.

### 1. 리브랜딩과 기술적 잔재의 정리
이번 업데이트에서 가장 눈에 띄는 부분은 리포지토리 명칭의 변경과 메시지입니다.

```diff
- "name": "cplabs_homepage",
+ "name": "metadium_homepage",
...
- "last_msg": "WEB2X 제목의 # 액센트 제거 (국/영문, 전 페이지 통일)",
+ "last_msg": "metadium 변환 5단계: cplabs 잔재 정리",
```

씨피랩스의 홈페이지를 메타디움(Metadium)으로 전환하는 과정은 단순한 텍스트 교체가 아닙니다. 브랜드의 정체성을 바꾸는 작업은 코드 곳곳에 숨어 있는 이전의 흔적들을 찾아내어 제거하는 '디지털 청소' 작업과 같습니다. 

특히 `cplabs 잔재 정리`라는 메시지에서 알 수 있듯, 대규모 리브랜딩 시에는 하드코딩된 문자열뿐만 아니라 설정 파일, 메타데이터, 그리고 내부 API 경로까지 꼼꼼하게 훑어야 합니다. 저는 이 과정을 '단계별 전환 전략'으로 접근하여, 시스템의 안정성을 해치지 않으면서 점진적으로 정체성을 교체하는 방식을 택했습니다.

### 2. SEO 최적화와 메타데이터 전략
또 다른 흥미로운 변경점은 씨피랩스 홈페이지의 SEO 강화 작업입니다.

```diff
+ "last_msg": "SEO: og:url·canonical 페이지별 세분화 (buildMetadata 헬퍼 도입)",
```

Web3와 블록체인 분야의 서비스는 기술적 완성도만큼이나 '발견 가능성(Discoverability)'이 중요합니다. 단순히 페이지를 만드는 것을 넘어, 검색 엔진이 각 페이지의 성격을 정확히 이해하도록 `canonical` 태그와 `og:url`을 세분화하는 작업이 필요했습니다.

여기서 제가 내린 기술적 결정은 **`buildMetadata`라는 헬퍼 함수를 도입**한 것입니다. 각 페이지마다 반복되는 메타데이터 생성 로직을 추상화함으로써, 코드 중복을 줄이고 유지보수 효율을 높였습니다. 이는 전형적인 '반복되는 패턴의 모듈화' 과정이며, 규모가 커질수록 이런 작은 설계의 차이가 개발 속도를 결정짓습니다.

### 3. AI 에이전트 연구: Claude-agent의 확장
개인적으로 집중하고 있는 `claude-agent` 리포지토리의 활동량 증가도 확인됩니다. 최근 Jira 프로젝트의 stale 이슈를 분석하거나 SSRF 차단 이슈를 해결하는 등, AI 에이전트가 실제 개발 워크플로우에 어떻게 깊숙이 관여할 수 있을지를 실험하고 있습니다. 

### 마치며: 기록이 주는 인사이트
오늘의 `dashboard.json` 업데이트는 단순한 숫자 변경이 아닙니다. 
- **씨피랩스**의 브랜드 전환이라는 비즈니스적 요구사항을 기술적으로 어떻게 풀어냈는지,
- **SEO 최적화**를 통해 서비스의 도달 범위를 어떻게 넓혔는지,
- **AI 에이전트**를 통해 엔지니어링 생산성을 어떻게 혁신할 것인지에 대한 고민의 흔적입니다.

코드 한 줄, 커밋 메시지 하나가 모여 결국 한 엔지니어의 성장 궤적을 만듭니다. 여러분도 오늘 여러분의 대시보드에는 어떤 기록이 남았나요?
