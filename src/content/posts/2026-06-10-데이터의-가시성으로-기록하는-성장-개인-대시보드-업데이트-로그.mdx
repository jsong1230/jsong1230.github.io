# [DevLog] 데이터의 흐름으로 기록하는 나의 성장, 대시보드 업데이트

안녕하세요, 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 **'가시성(Visibility)'**입니다. 시스템의 상태를 모니터링하는 것뿐만 아니라, 나 자신의 성장과 작업 흐름을 데이터로 시각화하는 것 역시 매우 중요하죠. 

오늘은 제 개인 대시보드의 데이터를 업데이트하며, 최근 제가 집중하고 있는 기술적 관심사와 작업 패턴을 되짚어보는 시간을 가졌습니다.

### 단순한 데이터 업데이트, 그 이상의 의미

오늘 진행한 작업은 표면적으로는 `dashboard.json` 파일의 데이터를 갱신하는 단순한 `chore` 작업입니다. 하지만 이 JSON 파일 하나에는 제가 최근 어떤 고민을 하고 어디에 시간을 쏟고 있는지가 고스란히 담겨 있습니다.

```json
{
  "name": "claude-agent",
  "last_msg": "logs: 2026-06-09 — GPU srv check, stolen-BTC signature verify, vanjeff node, k",
  "branch": "gram-jsong",
  "commits_7d": 1,
  "commits_30d": 2
}
```

특히 흥미로운 부분은 새롭게 추가된 `claude-agent` 저장소의 커밋 메시지입니다. **GPU 서버 체크, 탈취된 BTC의 서명 검증(stolen-BTC signature verify), 그리고 vanjeff 노드**와 같은 키워드들이 보입니다. 

최근 저는 AI 에이전트의 활용뿐만 아니라, 블록체인의 로우 레벨 분석과 AI 인프라의 효율적인 운용에 깊은 관심을 두고 있습니다. 특히 비트코인 서명 검증과 같은 작업은 Web3 전문가로서의 기술적 정체성을 유지하면서 AI라는 강력한 도구를 어떻게 접목할 것인가에 대한 제 개인적인 실험의 일환입니다.

### 기술적 결정과 인사이트: 정적인 데이터의 동적 관리

제 대시보드는 정적인 JSON 파일을 기반으로 작동합니다. 복잡한 DB를 구축할 수도 있었겠지만, 개인 포트폴리오 성격의 페이지에서는 **'단순함이 곧 효율성'**이라는 판단하에 Git 기반의 데이터 관리 방식을 선택했습니다.

이번 업데이트 과정에서 배운 점은 다음과 같습니다.

1. **데이터의 정렬과 구조화**: 기존에 흩어져 있던 저장소 순서를 조정하며, 현재 가장 활발하게 작업 중인 프로젝트(`jsong1230.github.io`, `company-page` 등)를 상단에 배치했습니다. 이는 단순한 정렬이 아니라, 제가 현재 우선순위를 어디에 두고 있는지를 스스로 정의하는 과정이기도 합니다.
2. **활동성(Activity)의 시각화**: `activity` 배열에 기록되는 일주일간의 커밋 수치를 통해, 특정 시점에 업무 부하가 어떻게 몰렸는지, 혹은 어느 시점에 기술적 탐색(Research) 시간이 많았는지를 직관적으로 파악할 수 있었습니다.

### 마치며: 씨피랩스에서의 경험을 개인의 자산으로

씨피랩스에서 CTO로서 수많은 프로젝트를 리딩하고 아키텍처를 설계하며 깨달은 점은, 결국 **'기록되지 않은 성장은 증명될 수 없다'**는 것입니다. 

비록 오늘 작업은 작은 JSON 파일의 수정이었지만, 이러한 작은 기록들이 모여 저의 기술적 궤적을 만들고, 미래의 저에게 중요한 인사이트를 제공할 것이라 믿습니다. 앞으로도 블록체인, AI, 그리고 DID 분야에서 끊임없이 탐구하고 그 과정을 투명하게 기록하겠습니다.

여러분은 자신의 성장을 어떻게 기록하고 계신가요?
