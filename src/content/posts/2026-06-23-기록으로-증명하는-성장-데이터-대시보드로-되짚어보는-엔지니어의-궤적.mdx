# [Dev Log] 데이터의 흐름으로 기록하는 나의 성장 궤적

안녕하세요, 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '기록'입니다. 단순히 코드를 짜는 것을 넘어, 내가 어떤 고민을 했고 어떤 리듬으로 움직였는지를 가시화하는 작업은 개인의 성장은 물론, 팀의 생산성을 관리하는 데에도 매우 중요한 지표가 됩니다.

오늘은 제 개인 대시보드의 데이터를 업데이트하며, 최근의 작업 흐름을 되짚어보는 시간을 가졌습니다.

### 단순한 업데이트, 그 이상의 의미

오늘 진행한 작업은 `dashboard.json` 파일의 데이터를 갱신하는 단순한 `chore` 작업이었습니다. 하지만 diff를 통해 살펴본 데이터의 변화는 제가 최근 어디에 집중하고 있는지를 명확히 보여줍니다.

```json
{
  "name": "heritageradio",
  "last_date": "2026-06-22",
  "last_msg": "[main] briefing: deploy 루트 .md sync gap — (a) 적용 완료 기록 (76568b",
  "branch": "dev",
  "commits_7d": 18,
  "commits_30d": 48
}
```

특히 `heritageradio` 프로젝트의 활동량이 눈에 띕니다. 최근 7일간 18개의 커밋이 발생했고, 배포 루트의 `.md` 싱크 갭을 해결하는 등 디테일한 정합성을 맞추는 작업에 집중했습니다. 인프라나 배포 파이프라인에서 발생하는 미세한 간극(gap)을 메우는 작업은 지루할 수 있지만, 시스템의 안정성을 결정짓는 결정적인 한 끗 차이가 됩니다.

### 기술적 결정과 인사이트: '가시화'의 힘

저는 씨피랩스에서 CTO로서 다양한 프로젝트를 리딩하며, 복잡한 시스템일수록 '현재 상태를 한눈에 볼 수 있는 대시보드'의 중요성을 체감했습니다. 이번에 제가 관리하는 `dashboard.json` 역시 그 연장선에 있습니다.

많은 개발자가 로그나 Git 히스토리를 통해 과거를 추적하지만, 이를 정형화된 데이터(`json`)로 변환하여 시각화하면 다음과 같은 인사이트를 얻을 수 있습니다.

1.  **컨텍스트 스위칭 비용 파악**: 여러 레포지토리를 오가며 작업할 때, 각 프로젝트에 투입되는 에너지의 분배를 객관적으로 볼 수 있습니다.
2.  **개발 리듬의 발견**: `activity` 배열에 기록된 일별 커밋 수를 통해 제가 언제 가장 몰입하는지, 혹은 언제 병목이 발생하는지 패턴을 분석할 수 있습니다.
3.  **우선순위 재검토**: `metadium_homepage`나 다른 프로젝트들의 활동량이 줄어든 것을 보며, 현재의 리소스 투입이 전략적 방향과 일치하는지 점검하게 됩니다.

### 마치며

코드 한 줄을 고치는 것보다 더 어려운 것은 '내가 무엇을 하고 있는지'를 정확히 정의하고 기록하는 일입니다. 씨피랩스에서의 경험을 통해 배운 것은, 최고의 엔지니어는 기술적 구현 능력뿐만 아니라 자신의 작업 프로세스를 데이터화하고 최적화할 줄 아는 사람이라는 점입니다.

오늘의 데이터 업데이트는 단순한 숫자 변경이 아니라, 다음 한 주를 설계하기 위한 일종의 '회고'였습니다. 여러분도 자신의 성장 궤적을 데이터로 기록해 보시는 건 어떨까요?
