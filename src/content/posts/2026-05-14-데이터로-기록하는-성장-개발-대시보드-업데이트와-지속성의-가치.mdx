## [Insight] 데이터로 기록하는 성장: 개발 대시보드 업데이트와 '지속성'의 가치

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

개발자에게 있어 '기록'은 단순한 메모 이상의 의미를 갖습니다. 내가 어떤 고민을 했고, 어떤 문제를 풀었으며, 시간이 흐름에 따라 나의 생산성이 어떻게 변화했는지를 객관적인 수치로 확인하는 과정은 성장을 위한 가장 강력한 피드백 루프가 되기 때문입니다.

오늘은 제가 개인적으로 운영하고 있는 개발 활동 대시보드의 데이터를 업데이트하며 느낀 몇 가지 인사이트를 공유해 보려 합니다.

### 단순한 데이터 업데이트, 그 이상의 의미

오늘 진행한 작업은 표면적으로는 `dashboard.json` 파일의 수치를 갱신하는 단순한 `chore` 작업이었습니다. 하지만 diff를 통해 살펴본 데이터의 변화에는 최근 저의 기술적 관심사와 작업 패턴이 고스란히 담겨 있습니다.

```json
- "last_msg": "[main] admin 인계 brief — § 2 expand: prod 잔재 누적 사고 + cleanup +",
+ "last_msg": "[landing+admin] flat 3 program 스크립트 클린업 — 인물 존칭 제거 +",
```

특히 눈에 띄는 부분은 `heritageradio` 프로젝트의 커밋 메시지 변화입니다. 이전 작업이 '프로덕션 잔재 제거'라는 인프라적 안정성에 집중했다면, 이번에는 '스크립트 클린업'과 '존칭 제거' 같은 세밀한 리팩토링 단계로 진입했음을 알 수 있습니다. 

큰 문제를 해결하는 '폭발적인 개발' 단계에서, 시스템을 정교하게 다듬는 '디테일한 최적화' 단계로 전환되는 과정은 시니어 엔지니어로서 가장 보람을 느끼는 지점이기도 합니다.

### 기술적 결정: 정적 데이터 기반의 대시보드

많은 분이 이런 대시보드를 구축할 때 실시간 API 연동을 생각하시겠지만, 저는 의도적으로 JSON 파일을 통한 정적 데이터 업데이트 방식을 유지하고 있습니다. 여기에는 두 가지 이유가 있습니다.

1. **성능과 비용의 최적화**: 굳이 매번 API를 호출해 렌더링할 필요 없이, 빌드 타임에 데이터를 확정 지음으로써 사용자(혹은 나 자신)에게 가장 빠른 응답 속도를 제공합니다.
2. **스냅샷의 가치**: 실시간 데이터는 '현재'를 보여주지만, 커밋으로 관리되는 JSON 데이터는 '역사'를 보여줍니다. Git history를 통해 내 활동량이 어떻게 변했는지 시계열로 분석할 수 있다는 점이 가장 큰 장점입니다.

### 잊지 말아야 할 '지속성'에 대하여

이번 업데이트에서 새롭게 추가된 `go-metadium` 프로젝트나, 꾸준히 숫자가 기록되고 있는 `jsong1230.github.io`의 activity 배열을 보며 다시금 깨닫습니다. 

Web3, DID, AI와 같이 변화 속도가 무시무시한 분야에서 살아남는 유일한 방법은 결국 **'매일 조금씩이라도 기록하고 움직이는 지속성'**입니다. 때로는 커밋 수가 적은 날도 있고, 정체된 것처럼 느껴지는 프로젝트도 있겠지만, 그 모든 데이터가 쌓여 결국 저라는 엔지니어의 정체성을 형성합니다.

여러분은 자신의 성장을 어떻게 기록하고 계신가요? 거창한 도구가 아니더라도 좋습니다. 나만의 '데이터 스냅샷'을 남기는 습관을 가져보시길 권합니다. 그것이 훗날 가장 강력한 포트폴리오가 될 것입니다.
