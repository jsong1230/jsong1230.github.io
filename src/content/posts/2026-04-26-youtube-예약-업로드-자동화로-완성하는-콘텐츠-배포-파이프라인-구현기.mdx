# [Dev Log] 자동화의 완성은 '디테일'에 있다: YouTube 예약 업로드 구현기

안녕하세요, 씨피랩스(CP Labs)의 송주한입니다.

엔지니어로서 제가 가장 좋아하는 순간은 반복적인 수작업을 코드로 대체하여 '시스템'으로 만드는 때입니다. 최근 저는 개인 프로젝트인 `booksummary` 서비스의 파이프라인을 고도화하고 있는데, 오늘은 그중에서도 콘텐츠 배포의 마지막 단계인 **YouTube 예약 업로드 자동화** 작업을 진행했습니다.

단순히 영상을 올리는 것을 넘어, '언제, 어떻게' 노출될 것인가라는 운영의 영역을 코드로 제어하는 과정에서 얻은 인사이트를 공유하고자 합니다.

### 1. 해결하고자 했던 문제: "업로드 후 수동 예약의 번거로움"

기존 파이프라인은 영상 생성부터 업로드까지 자동화되어 있었지만, 정작 가장 중요한 '공개 시간 설정'은 YouTube 스튜디오에 접속해 수동으로 처리해야 했습니다. 

콘텐츠 마케팅의 핵심은 일관된 시간대에 업로드하여 시청자의 습관을 만드는 것입니다. 저는 이를 위해 **"작업 다음 날 저녁 7시(KST)"**라는 구체적인 비즈니스 로직을 코드에 심기로 결정했습니다.

### 2. 기술적 결정과 구현 포인트

이번 작업의 핵심은 **타임존(Timezone) 처리**와 **API 제약 사항의 해결**이었습니다.

#### ① KST → UTC 변환의 정밀함
YouTube API는 표준 UTC 시간을 요구합니다. 한국 시간(KST) 기준 다음 날 19시를 정확히 계산하기 위해 `datetime`과 `timezone` 모듈을 활용했습니다.

```python
# 다음 날 19:00 KST (= 10:00 UTC) 예약 업로드 로직
kst = timezone(timedelta(hours=9))
now_kst = datetime.now(kst)
publish_kst = (now_kst + timedelta(days=1)).replace(hour=19, minute=0, second=0, microsecond=0)
publish_utc = publish_kst.astimezone(timezone.utc).strftime('%Y-%m-%dT%H:%M:%SZ')
```
단순히 시간을 더하는 것이 아니라, `replace`를 통해 분/초를 초기화함으로써 정확히 정각에 업로드되도록 설정했습니다.

#### ② API 제약 조건의 처리 (`privacyStatus`)
구현 중 흥미로웠던 점은 YouTube API의 특이한 제약 사항이었습니다. `publishAt` 파라미터를 사용하여 예약 업로드를 설정할 경우, `privacyStatus`는 반드시 `private` 상태여야 한다는 점입니다. 이를 간과하면 API 에러가 발생합니다.

```python
'status': {
    # publishAt 사용 시 YouTube API는 privacyStatus가 "private"이어야 함
    'privacyStatus': 'private' if publish_at else privacy_status,
    **({"publishAt": publish_at} if publish_at else {})
}
```
조건부 딕셔너리 언패킹(`**`)을 사용하여, 예약 시간이 있을 때만 `publishAt` 필드가 추가되고 상태값이 `private`로 강제되도록 처리했습니다.

### 3. 마치며: 데이터로 보는 성장

오늘 작업 후 제 개인 대시보드(`dashboard.json`)를 업데이트하며 최근의 활동량을 확인했습니다. `booksummary` 프로젝트에 새로운 기능이 추가되고, `heritageradio` 등 여러 프로젝트의 커밋 수치가 변하는 것을 보며 시스템이 살아 움직이고 있다는 느낌을 받습니다.

**"자동화의 끝은 단순히 시간을 줄이는 것이 아니라, 사람이 신경 써야 할 '결정의 영역'을 줄이는 것"**이라고 생각합니다. 이제 저는 "언제 올리지?"라는 고민 대신 "어떤 가치 있는 콘텐츠를 만들지?"에 더 집중할 수 있게 되었습니다.

앞으로도 씨피랩스에서의 경험과 개인적인 기술 탐구를 통해 더 효율적인 시스템 구축 방법을 계속해서 기록하겠습니다.
