# [Dev Log] 엣지 케이스의 발견: 한글 수사 처리와 정규표현식의 함정

안녕하세요, 씨피랩스의 CTO 송주한입니다.

개발자에게 가장 즐거우면서도 괴로운 순간은 언제일까요? 저는 '분명히 정상이라고 생각했는데, 데이터가 누락된 사각지대를 발견했을 때'라고 생각합니다. 오늘 저는 관리자 페이지의 시리즈 회차 추출 로직에서 바로 그 '사각지대'를 발견하고 해결하는 작업을 진행했습니다.

## 1. 문제의 발견: "왜 데이터가 안 보이지?"

최근 시리즈 데이터를 점검하던 중, 특정 프로그램들의 회차(Episode) 추출 결과가 0으로 나오거나 일부가 누락되는 현상을 발견했습니다. 로그를 뜯어보니 원인은 단순했지만 치명적인 **정규표현식의 맹점**에 있었습니다.

- **패턴의 경직성**: 기존에는 `^ep(\d+)`처럼 문자열 시작 부분에 `ep`가 오는 경우만 매칭했습니다. 하지만 실제 데이터에는 `"약이 되는 밥상 — ep06"`처럼 회차 정보가 끝에 붙는 경우가 많았습니다.
- **언어의 다양성**: `YASA_STORY` 같은 프로그램은 "제일화", "제십오화"처럼 한글 수사를 사용하고 있었습니다. 기존의 `\d+화` 패턴으로는 당연히 잡을 수 없는 영역이었죠.

## 2. 기술적 결정: 유연한 추출기와 한글 수사 파서

단순히 정규식을 몇 개 추가하는 것으로 끝낼 수도 있었지만, 앞으로의 확장성을 위해 `_extract_ep_number`라는 헬퍼 함수를 신설하여 추출 로직을 중앙 집중화했습니다.

### 핵심 변경 사항
가장 흥미로웠던 부분은 한글 수사를 정수로 변환하는 `_ko_numeral_to_int` 로직의 구현입니다.

```python
def _ko_numeral_to_int(s: str) -> int | None:
    """한글 수사 → 정수. 제일화=1 · 제십화=10 · 제십오화=15 · 제이십삼화=23 (1~99)."""
    if not s:
        return None
    total = 0
    if "백" in s:
        head, _, tail = s.partition("백")
        total += (_KO_DIGIT.get(head, 1)) * 100
        s = tail
    if "십" in s:
        head, _, tail = s.partition("십")
        total += (_KO_DIGIT.get(head, 1)) * 10
        total += _KO_DIGIT.get(tail, 0)
    else:
        total += _KO_DIGIT.get(s, 0)
    return total or None
```

단순한 매핑을 넘어 `partition`을 이용해 '백', '십' 단위의 자릿수를 계산하도록 설계했습니다. 덕분에 "제이십삼화" 같은 복합 수사도 정확하게 `23`이라는 정수로 변환할 수 있게 되었습니다. 또한, `^` 앵커를 제거하고 `.search()`를 사용하여 제목의 어느 위치에 있든 `ep##` 패턴을 찾아내도록 개선했습니다.

## 3. 결과와 인사이트

이번 수정으로 `YASA_STORY`의 추출 건수가 25건에서 40건으로 전량 복구되었고, 다른 프로그램들의 누락분도 모두 해결되어 전 프로그램 **Anomaly 0** 상태를 달성했습니다.

이번 작업을 통해 다시 한번 느낀 점은 **"데이터는 항상 개발자의 예상보다 지저분하다"**는 것입니다. 

1. **앵커(`^`, `$`) 사용의 주의**: 엄격한 검증이 필요한 경우가 아니라면, 사용자 입력 데이터나 타이틀 추출 시에는 유연한 매칭 전략이 필요합니다.
2. **도메인 특성 반영**: 한국어 서비스라면 아라비아 숫자뿐만 아니라 한글 수사 같은 언어적 특성이 데이터에 섞여 들어올 가능성을 항상 염두에 두어야 합니다.

마지막으로, 보안 취약점 해결을 위해 `npm audit fix`를 통해 `package-lock.json`의 의존성들을 최신 패치 버전으로 업데이트하며 오늘 작업을 마무리했습니다. 

작은 디테일이 모여 시스템의 완결성을 만든다는 믿음으로, 내일도 꼼꼼하게 코드를 짚어보겠습니다.
