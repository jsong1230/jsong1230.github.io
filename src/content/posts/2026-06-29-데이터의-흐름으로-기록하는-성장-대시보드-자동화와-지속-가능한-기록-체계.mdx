# [Dev Log] 데이터의 흐름으로 기록하는 성장: 대시보드 자동화와 지속적인 업데이트

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어에게 있어 '기록'은 단순히 과거를 남기는 것이 아니라, 현재의 생산성을 측정하고 미래의 방향성을 설정하는 중요한 도구입니다. 저는 개인적으로 제가 관리하는 여러 리포지토리의 활동량을 시각화하여 보여주는 대시보드를 운영하고 있습니다. 오늘 진행한 작업은 단순한 데이터 업데이트처럼 보이지만, 그 이면에는 '지속 가능한 기록 체계'에 대한 고민이 담겨 있습니다.

### 오늘의 작업: 대시보드 데이터 최신화 (`chore: Update dashboard data`)

오늘의 주요 작업은 `src/data/dashboard.json` 파일을 업데이트하여 최근의 개발 활동 지표를 반영하는 것이었습니다. 

```json
// 변경 전
"generated_at": "2026-06-27T23:00:01.817Z",
// 변경 후
"generated_at": "2026-06-28T23:00:01.825Z",
```

단순히 날짜가 하루 바뀌고 커밋 수가 조정된 것이지만, 이 데이터가 생성되는 과정에는 자동화된 파이프라인이 작동하고 있습니다. 특히 이번 업데이트에서 흥미로운 점은 `jsong1230.github.io` 리포지토리의 활동 변화입니다.

```json
{
  "name": "jsong1230.github.io",
  "last_msg": "Auto-generate weekly recap",
  "activity": [2, 2, 2, 2, 3, 3, 0],
  "commits_7d": 16,
  "commits_30d": 70
}
```

지난 30일간의 커밋 수가 69개에서 70개로 늘어났고, 최근 7일간의 활동 그래프(`activity`)가 업데이트되었습니다. 특히 `last_msg`에 적힌 **"Auto-generate weekly recap"**이라는 문구에서 알 수 있듯, 저는 기록의 자동화(Automation)를 통해 운영 공수를 줄이면서도 일관성을 유지하는 전략을 취하고 있습니다.

### 기술적 결정과 인사이트: 왜 '자동화된 기록'인가?

20년 넘게 엔지니어로 일하며 깨달은 점 중 하나는, 사람이 수동으로 작성하는 로그는 반드시 누락되거나 왜곡된다는 것입니다. 특히 바쁜 프로젝트 일정 속에서 "오늘 무엇을 했는가"를 매일 기록하는 것은 매우 고단한 일입니다.

그래서 저는 다음과 같은 기술적 방향성을 잡았습니다.

1. **데이터 기반의 객관화**: 주관적인 회고 이전에, Git 커밋 로그라는 객관적인 데이터를 먼저 수집합니다.
2. **파이프라인 구축**: 데이터 수집 $\rightarrow$ JSON 변환 $\rightarrow$ 대시보드 반영으로 이어지는 흐름을 자동화하여, 기록을 위한 기록이 아닌 '인사이트를 위한 기록'이 되게 합니다.
3. **시각적 피드백**: `activity` 배열과 같은 시계열 데이터를 통해 나의 집중도가 어느 시점에 높았는지, 어떤 프로젝트에 리소스를 더 투입했는지를 한눈에 파악합니다.

### 마치며

오늘의 `chore` 커밋은 규모가 작지만, 씨피랩스에서 복잡한 Web3나 AI 시스템을 설계할 때와 마찬가지로 **"작은 자동화가 모여 거대한 효율성을 만든다"**는 제 철학이 반영된 작업입니다.

단순한 숫자들의 나열처럼 보일지 모르지만, 이 데이터들은 제가 기술적으로 정체되지 않고 계속해서 움직이고 있다는 가장 정직한 증거가 됩니다. 여러분도 자신만의 '성장 지표'를 자동화하여 기록해 보시는 건 어떨까요?

내일은 더 깊이 있는 기술 이야기로 돌아오겠습니다.
