# [Dev Log] AI와 자동화로 구축하는 '생각의 아카이브': 개인 블로그 고도화 작업기

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한입니다.

엔지니어에게 블로그는 단순한 기록장을 넘어, 자신의 기술적 궤적을 증명하는 '생각의 아카이브'와 같습니다. 하지만 매일 쏟아지는 개발 작업과 인사이트를 꾸준히 정리하고, 이를 다시 체계적인 회고로 연결하는 것은 생각보다 많은 에너지가 드는 일입니다.

그래서 오늘은 제가 사용하는 개인 블로그(`jsong1230.github.io`)에 **AI 기반의 자동화 파이프라인**을 구축하고, 사용자 경험(UX)을 개선한 작업 내용을 공유하려 합니다.

---

### 1. "기록은 자동화하고, 통찰은 정제한다": 주간 회고 자동 생성
가장 핵심적인 작업은 `generate-weekly-recap.js` 스크립트를 구현한 것입니다. 매일 작성하는 데일리 포스트들을 AI가 분석해 한 주를 요약하는 '주간 회고'를 자동으로 생성하도록 만들었습니다.

**기술적 결정:**
- **로컬 LLM(Gemma) 활용:** 보안과 비용, 그리고 개인화된 튜닝을 위해 외부 API 대신 로컬 환경에서 구동되는 Gemma 모델을 연결했습니다.
- **파이프라인:** `포스트 수집` $\rightarrow$ `Gemma 기반 요약 및 인사이트 추출` $\rightarrow$ `MDX 파일 생성` $\rightarrow$ `Git Commit/Push` 순으로 이어지는 자동화 셸 스크립트를 구성했습니다.

특히 흥미로웠던 점은 AI에게 부여한 **페르소나 설정**입니다.
```javascript
const systemPrompt = isKorean
  ? `당신은 블록체인, Web3, DID, AI 분야의 전문가인 송주한(Jeffrey Joo-Han Song)입니다.
     한 주간의 개발 일지들을 모아 주간 회고 포스트를 작성합니다...`
  : `...`;
```
단순한 요약이 아니라, 저의 전문 분야(Web3, DID, AI)와 톤앤매너를 학습시킨 페르소나를 부여함으로써, AI가 작성한 초안 위에 저의 최종적인 통찰만 덧붙이면 되는 효율적인 워크플로우를 완성했습니다.

### 2. 시각적 정체성 확립: 디자인 시스템 및 OG 이미지 자동화
기술적 깊이만큼 중요한 것이 전달력입니다. 이번 업데이트에서는 디자인 시스템을 전면 수정하고, 포스트별 OG(Open Graph) 이미지를 자동 생성하는 기능을 추가했습니다.

- **디자인 시스템:** 'Warm Parchment' 팔레트와 세리프 타이포그래피를 적용해, 개발 블로그 특유의 딱딱함을 걷어내고 '읽기 편한 기록지'의 느낌을 주었습니다.
- **Satori 기반 OG 생성:** `src/pages/og/[slug].png.ts`를 통해 포스트의 제목과 메타데이터를 기반으로 이미지를 동적으로 생성합니다. 이제 링크를 공유할 때마다 각 포스트의 정체성이 담긴 이미지가 함께 노출됩니다.

### 3. 이번 작업을 통해 배운 점
이번 고도화 작업의 핵심은 **"반복되는 운영 비용의 최소화"**였습니다. 

많은 엔지니어가 기록의 중요성을 알면서도 중도에 포기하는 이유는 '정리'라는 운영 오버헤드가 크기 때문입니다. 저는 이를 AI 에이전트(Gemma + Scripts)에게 위임함으로써, 저는 오직 **'어떤 문제를 풀 것인가'**라는 본질적인 고민에만 집중할 수 있는 환경을 만들었습니다.

결국 Web3나 AI 시대의 생산성은 도구를 얼마나 잘 쓰느냐가 아니라, **나의 사고 과정을 얼마나 효율적인 파이프라인으로 설계하느냐**에 달려 있다는 것을 다시 한번 느꼈습니다.

앞으로도 씨피랩스에서 마주하는 기술적 도전들과 개인적인 탐구 과정을 이 자동화된 아카이브에 차곡차곡 쌓아나가겠습니다.
