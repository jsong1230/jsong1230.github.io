# [Dev Log] 운영 환경의 정교한 제어: Admin 화이트리스트와 FCM 인프라 구축

안녕하세요, 씨피랩스의 CTO 송주한입니다.

오늘은 프로젝트의 운영(Production) 환경을 활성화하고, 사용자 경험의 핵심인 푸시 알림 시스템을 구축하는 굵직한 작업들을 진행했습니다. 특히 운영 환경에서는 '편의성'보다 '보안'과 '정교한 제어'가 우선되어야 한다는 점을 다시 한번 체감한 하루였습니다.

## 1. 운영 Admin 활성화와 '화이트리스트'의 함정

당초 운영 환경의 Admin 페이지는 배포 일정에서 제외하려 했으나, 푸시 알림 전송과 피드백 확인 등 운영 필수 기능을 위해 활성화하기로 결정했습니다. 여기서 핵심은 **"개발 환경에서는 모든 관리 기능을 열어두되, 운영 환경에서는 허가된 특정 경로만 허용한다"**는 원칙이었습니다.

하지만 첫 배포 후 검증 과정에서 흥미로운 버그를 발견했습니다. `/admin` prefix를 사용해 화이트리스트를 관리하다 보니, `/admin/episode-registry` 같은 내부 편집 페이지까지 모두 통과되어 버리는 문제가 발생한 것입니다.

```typescript
// 문제의 원인: prefix 매칭으로 인해 /admin으로 시작하는 모든 경로가 통과됨
const PROD_ADMIN_ALLOWED_PREFIXES = [
  "/admin", // 이 설정이 모든 sub-path를 허용하는 결과를 초래
  "/admin/signin",
  // ...
];
```

**기술적 결정:**
단순히 `startsWith`로 체크하는 것이 아니라, **정확한 일치(Exact Match)**와 **접두사 일치(Prefix Match)**를 분리했습니다.

```typescript
// 수정 후: Root 대시보드는 정확히 일치할 때만, 나머지는 명시된 prefix만 허용
const PROD_ADMIN_ALLOWED_EXACT = new Set<string>([
  "/admin",
]);

function isProdAdminAllowed(pathname: string): boolean {
  if (PROD_ADMIN_ALLOWED_EXACT.has(pathname)) return true;
  return PROD_ADMIN_ALLOWED_PREFIXES.some(
    (p) => pathname === p || pathname.startsWith(p + "/") || pathname.startsWith(p + "?")
  );
}
```

작은 차이 같지만, 운영 환경에서의 권한 제어는 이처럼 '보수적'으로 접근해야 사고를 막을 수 있다는 인사이트를 다시금 얻었습니다.

## 2. Dev/Prod Parity: FCM 푸시 인프라 구축

다음으로 진행한 작업은 FCM(Firebase Cloud Messaging) 기반의 푸시 알림 시스템 구축입니다. 백엔드 서비스부터 모바일 클라이언트, 그리고 이를 비동기로 처리할 Celery 워커까지 전체 파이프라인을 연결했습니다.

여기서 제가 가장 중요하게 생각한 원칙은 **Dev/Prod Parity(개발-운영 환경 동일성)**입니다. 운영 환경에만 특수한 설정을 넣는 것이 아니라, 인프라 구성(CDK)과 배포 스크립트를 통해 동일한 아키텍처가 환경 변수만 바뀐 채 그대로 배포되도록 설계했습니다.

- **Backend:** `push_fcm.py` 서비스를 통해 메시지 발송 로직을 추상화하고, Celery를 통해 대량 발송 시의 병목을 방지했습니다.
- **Infra:** AWS Secrets Manager를 통해 OAuth 키와 Auth Secret을 안전하게 주입하고, ALB(Application Load Balancer)에서 SNI를 통해 `admin.radiotm.co.kr` 도메인을 직접 라우팅하도록 설정했습니다.

## 3. 마치며: 엔지니어의 디테일

오늘의 작업을 통해 느낀 점은, 시스템이 커질수록 코드를 짜는 시간보다 **'어떻게 안전하게 배포하고 제어할 것인가'**에 대한 고민의 시간이 더 길어진다는 것입니다. 

단순히 기능이 동작하는 것을 넘어, 잘못된 경로 접근을 차단하는 미들웨어의 정교함, 그리고 환경 간의 격차를 줄이는 인프라 자동화가 결국 서비스의 안정성으로 이어진다고 믿습니다. 씨피랩스에서도 이러한 엔지니어링 원칙을 바탕으로 더 단단한 제품을 만들어가겠습니다.
