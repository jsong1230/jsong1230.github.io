## [Dev Log] 데이터의 흐름으로 기록하는 나의 성장 궤적

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한입니다.

엔지니어로 20년 넘게 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '기록'입니다. 단순히 코드를 짜는 것을 넘어, 내가 어떤 고민을 했고 어떤 리듬으로 작업했는지를 데이터로 남기는 과정은 추후 기술적 의사결정을 복기할 때 매우 중요한 자산이 됩니다.

오늘 제가 진행한 작업은 거창한 아키텍처 설계는 아니었지만, 저의 개인 대시보드(`jsong1230.github.io`)의 데이터를 업데이트하는 작업이었습니다. 겉으로 보기에는 단순한 `json` 파일 수정처럼 보이지만, 그 이면에는 '엔지니어링 생산성 시각화'라는 저만의 작은 실험이 담겨 있습니다.

### 무엇을 해결하려 했는가?

저는 제가 관리하는 여러 레포지토리의 활동량(Commit 수, 최근 메시지, 브랜치 상태 등)을 자동으로 수집해 대시보드 형태로 관리합니다. 이번 업데이트에서는 특히 최근 집중하고 있는 프로젝트들의 활동성을 최신화하고, 정체된 프로젝트와 활발한 프로젝트를 구분하여 현재 나의 '기술적 관심사'가 어디로 흐르고 있는지를 확인하고자 했습니다.

### 코드 변경점과 인사이트

이번 `diff`에서 흥미로운 부분은 `heritageradio`와 `claude-agent` 레포지토리의 변화입니다.

```json
{
  "name": "heritageradio",
  "last_date": "2026-07-08",
  "last_msg": "[admin] briefing: w11 DRAMA_THEATER audio 재합성 요청 (정박사) — 1955/",
  "commits_7d": 4,
  "commits_30d": 33
}
```

특히 `claude-agent` 레포지토리의 커밋 메시지를 보면 최근 저의 관심사를 알 수 있습니다. `career history reconstruction from Gmail`과 같은 작업들이 포함되어 있는데, 이는 AI 에이전트를 활용해 파편화된 과거의 기록을 재구성하는 자동화 파이프라인을 구축하고 있다는 증거이기도 합니다.

단순히 `commits_7d`의 숫자가 올라가는 것보다 중요한 것은, **어떤 도메인의 문제를 풀기 위해 시간을 쓰고 있는가**를 데이터로 확인하는 것입니다. 

### 기술적 결정: 정적 데이터 기반의 대시보드

많은 분이 실시간 API 연동을 생각하시겠지만, 저는 개인 대시보드에 `dashboard.json`이라는 정적 파일을 활용하는 방식을 택했습니다. 이유는 명확합니다.

1. **성능과 비용**: 매번 GitHub API를 호출하는 것보다 빌드 타임에 데이터를 생성해 배포하는 것이 훨씬 빠르고 효율적입니다.
2. **불변성(Immutability)**: 특정 시점의 내 상태를 스냅샷 형태로 남길 수 있어, 시간이 흐른 뒤 "내가 이때는 이런 리듬으로 일했구나"라는 히스토리 추적이 가능합니다.

### 마치며

씨피랩스에서 수많은 복잡한 시스템을 설계하고 운영하고 있지만, 결국 모든 거대한 시스템의 시작은 이렇듯 작은 기록과 데이터의 축적에서 시작된다고 믿습니다. 

코드 한 줄, 커밋 메시지 하나가 모여 엔지니어의 정체성을 만듭니다. 여러분은 오늘 여러분의 성장 궤적을 어떻게 기록하고 계신가요?
