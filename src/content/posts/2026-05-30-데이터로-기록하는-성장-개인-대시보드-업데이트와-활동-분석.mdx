# [Dev Log] 데이터의 흐름으로 기록하는 나의 성장 궤적: 대시보드 업데이트

안녕하세요, 씨피랩스(CP Labs)의 CTO 송주한(Jeffrey)입니다.

엔지니어에게 있어 '기록'은 단순히 과거를 남기는 것이 아니라, 현재의 생산성을 측정하고 미래의 방향성을 설정하는 중요한 지표가 됩니다. 저는 개인적인 개발 활동과 프로젝트들의 진행 상황을 한눈에 파악하기 위해 개인 웹사이트(`jsong1230.github.io`)에 대시보드 기능을 구현해 두고 있습니다.

오늘은 이 대시보드의 핵심 데이터 소스인 `dashboard.json`을 업데이트하며, 최근의 개발 흐름을 정리하는 시간을 가졌습니다.

### 🛠 오늘의 작업: 데이터 동기화와 활동 분석

오늘의 작업은 단순한 `chore`성 업데이트처럼 보일 수 있지만, 그 내면에는 제가 관리하는 여러 프로젝트의 '건강 상태'를 체크하는 과정이 담겨 있습니다. 

이번 커밋에서 가장 눈에 띄는 변화는 프로젝트별 활동 지표의 변동입니다. 특히 `cpdax_admin` 프로젝트의 활동이 두드러졌습니다.

```json
// src/data/dashboard.json diff
- "name": "cp-check-attendance",
- "last_msg": "fix: compose 빌드 Dockerfile 경로 명시 및 deptApi 문법 에러 수정",
+ "name": "cpdax_admin",
+ "last_msg": "feat(urls): redirect root path to /admin/",
...
- "commits_7d": 2,
- "commits_30d": 2
+ "commits_7d": 5,
+ "commits_30d": 5
```

최근 씨피랩스에서 진행 중인 관리자 페이지(`cpdax_admin`)의 라우팅 최적화 작업과 루트 경로 리다이렉션 설정 등, 사용자 경험(UX)을 개선하기 위한 세밀한 튜닝 작업들이 반영되었습니다. 작은 변경사항 같지만, 엔터프라이즈 급 서비스에서는 이러한 경로 설정 하나가 인프라의 Nginx 설정과 맞물려 전체 서비스의 안정성을 결정짓기 때문에 매우 신중하게 접근하고 있습니다.

### 💡 기술적 인사이트: 정적 데이터의 동적 활용

저는 왜 굳이 JSON 파일을 통해 데이터를 관리하고 이를 커밋하는 방식을 택했을까요? 

1. **불변성과 추적 가능성**: DB를 사용하지 않고 Git 저장소에 데이터를 기록함으로써, 내 개발 생산성이 시간에 따라 어떻게 변했는지 `git log`를 통해 완벽하게 추적할 수 있습니다.
2. **제로 비용의 인프라**: GitHub Pages라는 정적 호스팅 환경에서 별도의 서버 없이도 나만의 '활동 분석 툴'을 운영할 수 있다는 점이 매력적입니다.
3. **자동화의 가치**: 이번 업데이트에서도 볼 수 있듯이 `Auto-generate daily posts`와 같은 파이프라인을 통해 데이터가 자동으로 갱신되도록 설계했습니다. 이는 "반복되는 단순 작업은 반드시 자동화한다"는 저의 엔지니어링 철학이 반영된 결과입니다.

### 🚀 마치며

20년 넘게 엔지니어로 살아오며 느낀 점은, 결국 '측정할 수 없는 것은 개선할 수 없다'는 것입니다. 코드 한 줄을 더 짜는 것만큼이나, 내가 어디에 시간을 쓰고 있고 어떤 프로젝트가 정체되어 있는지를 데이터로 확인하는 과정이 중요합니다.

오늘 업데이트한 대시보드의 수치들이 단순한 숫자가 아니라, 씨피랩스에서 만들어가는 기술적 가치와 개인적인 성장의 증거가 되길 바랍니다.

내일은 또 어떤 문제를 풀게 될까요? 기록은 계속됩니다.
