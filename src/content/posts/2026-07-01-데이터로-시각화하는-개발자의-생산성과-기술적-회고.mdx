# [DevLog] 데이터가 말해주는 개발자의 리듬: 대시보드 업데이트와 회고

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

엔지니어로서 매일 코드를 작성하고 시스템을 설계하는 일도 중요하지만, 제가 개인적으로 중요하게 생각하는 루틴 중 하나는 바로 **'나의 생산성을 시각화하는 것'**입니다. 오늘은 제 개인 개발 대시보드의 데이터를 업데이트하며, 지난 며칠간의 작업 흐름을 되짚어보는 시간을 가졌습니다.

단순히 `dashboard.json` 파일의 숫자를 바꾸는 작업처럼 보일 수 있지만, 이 과정은 저에게 일종의 '기술적 회고'와 같습니다.

### 📈 데이터로 보는 최근의 관심사

이번 업데이트에서 가장 눈에 띄는 부분은 `heritageradio`와 `mirae` 프로젝트의 활성도입니다.

```json
{
  "name": "mirae",
  "last_date": "2026-06-30",
  "last_msg": "feat: SOL TOP2 변동성 그리드 알림 (grid_alert.sh) + 백테스트",
  "branch": "main",
  "activity": [2, 0, ...]
}
```

최근 솔라나(SOL) 생태계의 변동성을 모니터링하고, 이를 효율적으로 알림 받을 수 있는 그리드 시스템을 구축하는 데 집중했습니다. 특히 `grid_alert.sh`와 같은 쉘 스크립트를 통한 자동화와 백테스트 과정을 거치며, 복잡한 프레임워크 없이도 핵심 로직을 빠르게 검증하는 '린(Lean)한 개발'의 가치를 다시금 느꼈습니다.

또한 `metadium_homepage` 작업에서는 모바일 Hero 슬라이더의 반응형 수정 작업을 진행했습니다. Web3 서비스일수록 사용자 경험(UX)의 진입장벽을 낮추는 것이 중요한데, 아주 작은 CSS 수정 하나가 전체 서비스의 완성도를 결정짓는다는 점을 매번 배웁니다.

### 🛠️ 기술적 결정과 인사이트: 자동화의 힘

제가 운영하는 이 대시보드는 수동으로 입력하는 것이 아니라, 특정 스크립트를 통해 GitHub API에서 커밋 로그와 활동 데이터를 가져와 JSON 형태로 자동 생성합니다.

이번 커밋 메시지인 `chore: Update dashboard data`는 단순해 보이지만, 그 이면에는 **"측정할 수 없으면 개선할 수 없다"**는 철학이 담겨 있습니다. 7일 및 30일 단위의 커밋 수(`commits_7d`, `commits_30d`)를 추적함으로써, 제가 현재 어떤 도메인(AI, 블록체인, 인프라 등)에 에너지를 쏟고 있는지 객관적으로 파악할 수 있기 때문입니다.

### 💡 마치며: 엔지니어의 성장은 기록에서 온다

20년 넘게 엔지니어로 일하며 깨달은 점은, 뛰어난 개발자는 단순히 코드를 잘 짜는 사람이 아니라 **자신의 작업 패턴을 이해하고 최적화하는 사람**이라는 것입니다. 

씨피랩스에서 수많은 프로젝트를 리딩하며 팀원들에게 항상 강조하는 것도 '기록'과 '가시화'입니다. 오늘처럼 짧은 데이터 업데이트만으로도 내가 어디서 정체되었고, 어디서 몰입했는지를 알 수 있다면 그것이 바로 성장의 발판이 됩니다.

여러분은 자신의 성장 궤적을 어떻게 기록하고 계신가요? 때로는 복잡한 툴보다, 나만의 단순한 JSON 파일 하나가 더 명확한 답을 줄 때가 있습니다.
