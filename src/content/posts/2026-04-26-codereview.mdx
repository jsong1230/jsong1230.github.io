안녕하세요. 시니어 소프트웨어 엔지니어이자 코드 리뷰어입니다.

오늘 제출하신 커밋들은 크게 **대시보드 데이터 업데이트**와 **YouTube 예약 업로드 기능 구현** 두 가지 작업으로 나뉩니다. 전자는 단순 데이터 갱신이지만, 후자는 외부 API의 제약 사항을 정확히 파악하고 비즈니스 로직에 반영한 실용적인 구현이 돋보입니다.

상세 리뷰 결과입니다.

---

## ✅ 잘한 점

1. **외부 API 제약 사항의 정확한 처리 (`src/09_upload_from_metadata.py`)**
   - YouTube API의 특성상 `publishAt`을 설정할 때 `privacyStatus`가 반드시 `private`이어야 한다는 제약 조건을 정확히 파악하고, `'private' if publish_at else privacy_status`와 같이 조건부 로직으로 처리한 점이 매우 훌륭합니다. 이는 런타임 에러를 사전에 방지하는 견고한 구현입니다.

2. **명확한 타임존 처리 및 표준 포맷 준수 (`src/08_create_and_preview_videos.py`)**
   - KST(UTC+9) 기준 다음 날 19:00라는 비즈니스 요구사항을 `timezone`과 `timedelta`를 사용하여 명시적으로 계산했습니다. 특히 API 전송을 위해 최종적으로 `astimezone(timezone.utc)`를 통해 ISO 8601 표준 포맷(`%Y-%m-%dT%H:%M:%SZ`)으로 변환한 과정이 매우 정확합니다.

3. **데이터 일관성 유지 (`src/data/dashboard.json`)**
   - 단순 데이터 업데이트임에도 불구하고 `generated_at` 필드를 최신화하여, 데이터의 최신성(Freshness)을 추적할 수 있도록 관리하고 있습니다.

## 🔧 개선할 점

1. **중복 코드의 함수화 필요 (`src/08_...` 및 `src/20_...`)**
   - `08_create_and_preview_videos.py`와 `20_create_episode_metadata.py` 두 곳에서 **예약 시간을 계산하는 로직이 완전히 동일하게 중복**되어 있습니다.
   - **개선 제안:** `src/utils/datetime_helper.py`와 같은 공통 모듈을 만들어 `get_next_day_scheduled_time(hour=19)` 형태의 함수로 추출하세요. 유지보수 시 예약 시간을 변경해야 한다면 현재 구조에서는 두 파일을 모두 수정해야 하며, 이는 실수로 인한 불일치를 유발할 수 있습니다.

2. **함수 내부 Import 지양**
   - 현재 `save_metadata`와 `create_episode_metadata` 함수 내부에서 `from datetime import timezone, timedelta`를 수행하고 있습니다.
   - **개선 제안:** 파이썬 스타일 가이드(PEP 8)에 따라 모든 import는 파일 최상단에 위치시키는 것이 좋습니다. 함수 내부 import는 순환 참조 해결이나 매우 무거운 라이브러리를 지연 로딩할 때만 제한적으로 사용하시길 권장합니다.

3. **하드코딩된 예약 시간의 설정값 분리**
   - `hour=19`라는 값이 코드 내에 하드코딩되어 있습니다.
   - **개선 제안:** 향후 업로드 전략이 변경될 가능성이 높으므로, `.env` 파일이나 `config.py`에 `YOUTUBE_SCHEDULE_HOUR = 19`와 같이 상수로 관리하는 것이 더 유연한 설계입니다.

## 💡 오늘의 기술 포인트

**[Timezone-Aware Datetime Handling]**
오늘 작업의 핵심은 **"로컬 비즈니스 시간(KST) $\rightarrow$ 표준 시간(UTC) $\rightarrow$ API 포맷(ISO 8601)"**으로 이어지는 시간 변환 파이프라인입니다. 

특히 `datetime.now(kst)`를 통해 처음부터 Timezone-aware 객체를 생성하고, `replace`를 통해 특정 시점을 지정한 뒤 `astimezone`으로 변환하는 흐름은 시간 관련 버그를 최소화하는 정석적인 방법입니다. 분산 환경이나 글로벌 서비스에서 매우 중요한 역량입니다.
