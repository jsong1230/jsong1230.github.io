# [Dev Log] 데이터의 흐름으로 기록하는 나의 성장: 대시보드 업데이트와 회고

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

엔지니어로서 커리어를 쌓아오며 제가 가장 중요하게 생각하는 것 중 하나는 '가시화(Visualization)'입니다. 아무리 많은 코드를 작성하고 복잡한 아키텍처를 설계하더라도, 그것이 어떤 흐름으로 진행되고 있는지 데이터로 확인하지 못하면 개선의 방향을 잡기 어렵기 때문입니다.

오늘은 제 개인 대시보드의 데이터를 업데이트하며, 지난 며칠간의 개발 활동을 되돌아보는 시간을 가졌습니다. 단순한 `chore` 작업처럼 보일 수 있는 JSON 파일 수정이지만, 그 속에 담긴 수치들은 저에게 많은 인사이트를 줍니다.

### 📊 데이터가 말해주는 것들

오늘 업데이트한 `dashboard.json`의 diff를 보면 여러 레포지토리의 활동성(activity)과 커밋 수의 변화가 나타납니다.

```json
- "commits_7d": 34,
+ "commits_7d": 14,
```

특히 눈에 띄는 부분은 `company-page` 레포지토리의 7일간 커밋 수 변화입니다. 34건에서 14건으로 급격히 줄어든 것을 볼 수 있습니다. 이는 단순히 작업량이 줄어든 것이 아니라, 집중적인 기능 구현 단계(Implementation phase)를 지나 안정화 및 검토 단계로 진입했음을 의미합니다. 

또한 `cp-check-attendance`와 같은 씨피랩스의 프로젝트들에서도 30일 기준 커밋 수의 변동이 보입니다. 이는 프로젝트의 생애주기에 따라 리소스 투입량이 유동적으로 변하는 자연스러운 과정입니다.

### 💡 기술적 결정과 인사이트: 왜 '자동화된 기록'인가?

저는 개인적인 생산성 관리를 위해 이러한 대시보드 시스템을 구축해 운영하고 있습니다. 매번 수동으로 기록하는 것은 불가능에 가깝기에, 특정 스크립트를 통해 GitHub API에서 데이터를 가져와 JSON 형태로 저장하고 이를 정적 페이지로 렌더링하는 방식을 택했습니다.

여기서 제가 얻은 기술적 인사이트는 **"측정할 수 없으면 관리할 수 없다"**는 것입니다. 

1. **객관적 회고**: "요즘 바쁘다"라는 막연한 느낌 대신, `commits_7d`라는 구체적인 지표를 통해 제가 어디에 시간을 쏟았는지 객관적으로 파악할 수 있습니다.
2. **컨텍스트 스위칭의 기록**: 여러 프로젝트(씨피랩스 업무, 개인 연구, 오픈소스 등)를 동시에 진행하는 CTO의 특성상, 어떤 프로젝트가 현재 'Hot'한 상태인지 한눈에 파악하는 것은 효율적인 시간 배분을 가능하게 합니다.
3. **지속성의 증명**: 매일의 activity 배열(`[3, 2, 2, 2, 2...]`)을 채워나가는 과정은 단순한 기록을 넘어, 엔지니어로서의 성실함과 지속적인 학습 태도를 유지하게 만드는 강력한 동기부여가 됩니다.

### 마치며

오늘의 작업은 단순한 데이터 업데이트였지만, 이를 통해 제가 걷고 있는 방향을 다시 한번 점검할 수 있었습니다. 블록체인, Web3, AI와 같이 빠르게 변하는 기술 생태계에서 중심을 잡는 방법은 결국 '기록'과 '분석'에 있다고 믿습니다.

내일은 또 어떤 데이터가 제 대시보드를 채우게 될까요? 더 나은 코드를 작성하고, 더 효율적인 시스템을 설계하는 내일을 기대하며 오늘의 로그를 마칩니다.
