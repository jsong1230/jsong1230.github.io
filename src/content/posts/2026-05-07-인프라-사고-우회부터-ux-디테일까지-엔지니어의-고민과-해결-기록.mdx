# 인프라의 '사고 우회'와 UX의 '디테일': 오늘의 개발 기록

안녕하세요, 씨피랩스의 CTO 송주한입니다.

오늘은 인프라의 치명적인 병목을 해결하는 '우회 전략'부터 사용자가 느끼는 아주 작은 시각적 불일치를 해결하는 '디테일'까지, 폭넓은 스펙트럼의 작업들을 진행했습니다. 엔지니어로서 때로는 거대한 아키텍처를 설계하지만, 때로는 아이콘 하나가 주는 사용자 경험(UX)에 집착해야 하는 것이 저희의 숙명인 것 같습니다.

오늘 제가 고민하고 해결한 핵심 포인트들을 공유합니다.

### 1. 인프라 사고 우회: OAC 이슈와 `hls_proxy` 도입
가장 긴박했던 작업은 CloudFront의 OAC(Origin Access Control) 설정 과정에서 발생한 사고를 우회하는 것이었습니다. 특정 조건에서 `/live/*` 경로의 접근 제어가 예상과 다르게 작동하며 서비스 가용성에 영향을 줄 수 있는 상황이었습니다.

여기서 제가 내린 기술적 결정은 **'직접 접근' 대신 '프록시를 통한 스트리밍'**이었습니다. 

- **해결책**: `backend/app/routers/hls_proxy.py`를 새로 구현하여 CloudFront가 직접 S3나 오리진에 접근하는 대신, 백엔드 서버가 `boto3`를 통해 데이터를 스트림으로 쏘아주는 방식으로 변경했습니다.
- **인사이트**: 이론적으로는 OAC 설정을 완벽하게 잡는 것이 정석이지만, 실서비스 운영 중에는 '정답'보다 '가용성'이 우선입니다. 빠르게 우회 경로를 확보하고, 이후 안정적인 시점에 다시 정석적인 설정을 시도하는 전략을 취했습니다.

### 2. 안정성을 위한 장치들: Exponential Backoff와 LRU Cache
네트워크 환경은 언제나 불안정합니다. 특히 모바일 환경에서의 HLS 재생은 더 그렇습니다.

- **Mobile**: `PlaybackError` 발생 시 단순히 재시도하는 것이 아니라, **Exponential Backoff(지수 백오프)** 전략을 도입했습니다. 네트워크 일시 오류 시 서버에 과부하를 주지 않으면서 자동으로 복구될 수 있는 시간을 벌어주는 장치입니다.
- **Backend**: `hls_builder`에서 `/tmp/audio_cache`가 무한정 커지는 것을 막기 위해 **LRU(Least Recently Used) cleanup** 로직을 추가하고 3GB 캡을 씌웠습니다. 인프라 비용 최적화와 디스크 풀(Full) 사고를 방지하기 위한 필수적인 조치였습니다.

### 3. UX의 완성은 '일관성': 아이콘의 의미론적 통일
기술적인 큰 산들을 넘고 나서 살펴본 것은 GNB(Global Navigation Bar)의 테마 토글 아이콘이었습니다. 기존에는 현재 상태를 보여주는 방식이었는데, 이를 **"클릭 시 일어날 다음 동작"**으로 통일했습니다.

```typescript
// AS-IS: 현재가 light면 해(Sun)를 보여줌
{theme === "light" ? <SunIcon /> : <MoonIcon />}

// TO-BE: 현재가 light면 '다크로 전환'한다는 의미의 달(Moon)을 보여줌
{theme === "light" ? <MoonIcon /> : <SunIcon />}
```

작은 변경 같지만, 사용자는 "지금 내 상태가 무엇인가"보다 "이 버튼을 누르면 어떻게 되는가"를 더 직관적으로 인지합니다. 서비스 전체에서 이 규칙을 통일함으로써 인지 부하를 줄였습니다.

### 마치며
오늘의 작업을 한 문장으로 정리하자면 **"거시적인 가용성 확보와 미시적인 일관성 부여"**라고 할 수 있겠네요. 

인프라 구성도를 HTML/Mermaid로 시각화하며 전체 흐름을 다시 정리한 것처럼, 복잡한 시스템일수록 문서화와 시각화가 중요합니다. 내일은 오늘 구축한 우회 경로를 어떻게 더 효율적으로 최적화할지 고민해봐야겠습니다.
