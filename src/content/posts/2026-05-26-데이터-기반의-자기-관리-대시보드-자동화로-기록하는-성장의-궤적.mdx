# 데이터가 말해주는 성장의 궤적: 대시보드 자동화와 기록의 가치

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어에게 있어 '기록'은 단순히 과거를 남기는 행위가 아니라, 자신의 성장 궤적을 추적하고 병목 지점을 찾아내는 중요한 데이터 분석 과정입니다. 저는 개인적인 생산성 관리와 기술적 성찰을 위해 저만의 개발 대시보드를 운영하고 있습니다. 

오늘 진행한 작업은 단순해 보이지만, 제가 지향하는 '데이터 기반의 자기 관리' 철학이 담긴 `dashboard.json` 데이터 업데이트 작업이었습니다.

### 단순한 업데이트, 그 이상의 의미

오늘의 커밋 내역을 보시면 `chore: Update dashboard data`라는 짧은 메시지와 함께 JSON 파일의 수치들이 변경된 것을 보실 수 있습니다. 

```json
- "last_msg": "Auto-generate weekly recap",
+ "last_msg": "Auto-generate daily posts",
...
- "commits_7d": 16,
- "commits_30d": 69
+ "commits_7d": 19,
+ "commits_30d": 72
```

단순히 숫자가 바뀌는 작업처럼 보이지만, 여기에는 두 가지 기술적/전략적 결정이 숨어 있습니다.

**첫째, '자동화 파이프라인'의 구축입니다.** 
매번 수동으로 커밋 수를 세어 입력하는 것은 엔지니어답지 않은 일입니다. 저는 GitHub API를 활용해 제가 관리하는 여러 레포지토리의 활동량(activity), 최근 커밋 메시지, 기간별 커밋 횟수를 자동으로 수집하여 JSON으로 생성하는 파이프라인을 구축해 두었습니다. 오늘 업데이트된 `generated_at` 타임스탬프가 그 결과물입니다.

**둘째, '가시성(Visibility)'의 확보입니다.** 
`heritageradio`나 `trump-code` 같은 프로젝트들의 활동 지표를 한눈에 보며, 제가 현재 어떤 도메인에 에너지를 쏟고 있는지, 혹은 방치되고 있는 프로젝트는 없는지 객관적으로 파악합니다.

### 코드 변경에서 얻은 인사이트: 꾸준함의 힘

이번 diff에서 흥미로운 점은 `jsong1230.github.io` 레포지토리의 `commits_7d` 수치가 16에서 19로 상승했다는 점입니다. 특히 `activity` 배열의 수치 변화를 보면 특정 요일에 집중된 작업량이 보입니다.

20년 넘게 엔지니어로 일하며 깨달은 점은, 폭발적인 단기 집중력보다 더 무서운 것이 '작은 단위의 꾸준한 기록'이라는 것입니다. 대시보드에 찍히는 숫자 하나하나가 결국 제가 씨피랩스에서 이끄는 기술적 방향성과 개인적인 연구 성과로 이어지는 기초 체력이 됩니다.

### 마치며

많은 분이 "바쁜 업무 중에 언제 그런 기록까지 하느냐"고 묻습니다. 하지만 저는 오히려 기록하기 때문에 시간을 효율적으로 쓸 수 있다고 믿습니다. 내가 어디에 시간을 썼는지 데이터로 증명되면, 불필요한 루틴을 제거하고 더 본질적인 아키텍처 고민과 AI, Web3 같은 미래 기술 연구에 몰입할 수 있기 때문입니다.

여러분의 대시보드에는 오늘 어떤 기록이 남았나요? 숫자가 주는 정직한 피드백을 통해 내일의 성장 동력을 찾아보시길 바랍니다.
