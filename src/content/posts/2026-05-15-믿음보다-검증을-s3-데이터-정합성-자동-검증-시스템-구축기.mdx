# [Dev Log] "믿음"보다는 "검증"을: S3 동기화 정합성 자동 검증 시스템 구축기

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey Joo-Han Song)입니다.

엔지니어로서 가장 긴장되는 순간 중 하나는 '배포 후 데이터가 제대로 들어갔는지'를 확인할 때입니다. 특히 대량의 미디어 파일(HLS ts/m3u8)을 다루는 환경에서는 단순히 "업로드 성공" 메시지만으로는 안심할 수 없습니다. 네트워크 일시 오류나 예기치 못한 파일 손상으로 인해 실제 프로덕션 환경에 '깨진 파일'이 올라갔을 때, 이를 뒤늦게 발견하면 사용자 경험에 치명적인 영향을 주기 때문입니다.

오늘은 이러한 불안함을 제거하기 위해 도입한 **'Byte-identical 검증 시스템'**에 대해 이야기해보려 합니다.

### 1. 문제 정의: "업로드가 됐는데, 정말 똑같은 파일인가?"

기존의 동기화 스크립트는 파일 전송 후 성공 여부만 판단하고 바로 CloudFront Invalidation(캐시 무효화)을 진행했습니다. 하지만 전송 과정에서 아주 미세한 바이트 차이가 발생하거나, 소스 파일이 변경된 상태에서 덮어쓰기가 되었을 때 이를 감지할 방법이 없었습니다.

저는 단순히 "파일이 존재한다"는 확인을 넘어, **개발 서버의 아카이브 파일과 프로덕션 S3의 파일이 비트 단위로 완벽히 일치하는지**를 자동으로 검증하는 프로세스가 필요하다고 판단했습니다.

### 2. 기술적 결정: MD5와 S3 ETag의 활용

모든 파일을 다시 다운로드하여 비교하는 것은 비용과 시간 면에서 비효율적입니다. 여기서 저는 S3의 **ETag** 특성을 활용했습니다.

S3에서 Single-part upload로 올라간 파일의 ETag는 해당 파일의 **MD5 해시값**과 동일합니다. 우리가 다루는 `.ts`나 `.m3u8` 파일들은 크기가 매우 작기 때문에(약 110KB ~ 280KB) 모두 Single-part upload 조건에 해당하며, 따라서 `ETag == MD5`라는 가정이 성립합니다.

이를 바탕으로 다음과 같은 로직의 `verify_prod_sync.py`를 구현했습니다.
- **개발 서버:** 로컬 파일의 MD5 해시 생성
- **프로덕션 S3:** 파일의 ETag 값 추출
- **비교:** 두 값이 일치하는지 확인 $\rightarrow$ 불일치 시 즉시 에러 발생

### 3. 코드 변경과 안전장치

가장 핵심적인 변경 사항은 동기화 스크립트 내에 검증 단계를 삽입하고, **검증 실패 시 후속 작업을 차단**하는 것입니다.

```bash
# verify_prod_sync.py 호출 부분
if python3 /home/jsong/heritageradio/scripts/verify_prod_sync.py \
    --date "$TARGET_FULL" --cycles "01" --quiet > "$VERIFY_OUT" 2>&1; then
  VERIFY_RESULT=$(tail -1 "$VERIFY_OUT")
  echo "verify OK: $VERIFY_RESULT"
else
  # 검증 실패 시: Invalidation 스킵 + 텔레그램 알림 + 헬스체크 실패 처리
  tg_send "❌ <b>cycle 1 verify 실패... ⚠️ invalidation skip"
  hc_ping /fail
  exit 1
fi
```

여기서 주목할 점은 **`invalidation skip`**입니다. 검증에 실패했다는 것은 프로덕션에 잘못된 데이터가 올라갔음을 의미합니다. 이때 캐시를 무효화하면 사용자는 즉시 '깨진 파일'을 보게 됩니다. 따라서 검증 실패 시에는 캐시 무효화를 중단하여, 기존의 (비록 오래되었을지언정) 정상적인 캐시 데이터를 유지하게 함으로써 서비스 가용성을 확보했습니다.

### 4. 인사이트: 자동화된 신뢰의 가치

이번 작업을 통해 느낀 점은 **"신뢰는 자동화된 검증에서 온다"**는 것입니다. 

수동으로 확인했다면 놓쳤을 미세한 불일치를 시스템이 24시간 감시하게 함으로써, 운영자는 더 이상 "제대로 올라갔겠지"라고 추측할 필요가 없습니다. 특히 텔레그램 알림과 헬스체크(`hc_ping /fail`)를 연동하여 장애 인지 시간을 최소화한 점이 만족스럽습니다.

기술적인 깊이는 때로 아주 단순한 원리(MD5 비교)를 적재적소에 적용하는 것에서 시작됩니다. 앞으로도 씨피랩스에서 더 견고하고 신뢰할 수 있는 인프라를 구축해 나가겠습니다.
