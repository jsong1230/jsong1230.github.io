# [Dev Log] 사소한 설정 변경이 가져오는 운영의 디테일: 인프라 최적화와 사용자 경험

안녕하세요, 씨피랩스의 CTO 송주한(Jeffrey)입니다.

엔지니어로서 커리어를 쌓으며 깨달은 점 중 하나는, 거대한 아키텍처 설계만큼이나 중요한 것이 바로 '디테일한 설정'이라는 점입니다. 아주 작은 설정 오류 하나가 디버깅에 수 시간을 쏟게 만들기도 하고, 단순한 리다이렉트 하나가 사용자의 진입 장벽을 낮추기도 하죠.

오늘은 씨피랩스에서 진행 중인 `cpdax_admin` 프로젝트의 운영 효율을 높이기 위한 몇 가지 수정 사항과 개인 대시보드 업데이트 내용을 정리해 보려 합니다.

### 1. 404 페이지 대신 '정답'을 제시하는 UX
관리자 페이지를 운영하다 보면, 도메인 루트(`/`)로 접속했을 때 아무런 설정이 없어 404 Not Found 페이지가 뜨는 경우가 많습니다. 개발자에게는 익숙한 화면이지만, 운영자나 사용자 입장에서는 "사이트가 죽었나?"라는 오해를 불러일으킬 수 있죠.

이를 해결하기 위해 Django의 `RedirectView`를 활용해 루트 경로 접속 시 자동으로 `/admin/`으로 리다이렉트되도록 수정했습니다.

```python
# CPTM/urls.py
from django.views.generic import RedirectView

urlpatterns = [
    path('', RedirectView.as_view(url='/admin/', permanent=False)),
    path('admin/', admin.site.urls),
    # ...
]
```

단순한 두 줄의 코드 추가지만, 서비스의 완성도를 결정짓는 것은 이런 작은 배려에서 시작된다고 생각합니다.

### 2. 인프라 설정의 함정: Tailscale IP 수정
오늘 가장 뼈아픈(?) 실수는 네트워크 설정에서 나왔습니다. Tailscale을 이용해 가상 사설망을 구축해 사용하고 있는데, `settings.py`의 `ALLOWED_HOSTS`에 서버 IP가 아닌 제 개인 노트북(gram-jsong)의 IP를 적어두었더군요.

*   **기존:** `100.86.107.38` (노트북 IP) $\rightarrow$ **수정:** `100.80.56.102` (실제 서버 IP)

인프라 작업 시 IP 주소 하나, 오타 하나가 통신 단절로 이어지는 경험은 베테랑 엔지니어에게도 늘 긴장감을 줍니다. 이번 수정과 함께 Route53 A 레코드까지 일치시켜 네트워크 경로를 명확히 정리했습니다.

### 3. 데이터로 보는 성장: 대시보드 업데이트
마지막으로 개인 깃허브 대시보드(`jsong1230.github.io`)의 데이터를 업데이트했습니다. `dashboard.json` 파일을 통해 최근 7일 및 30일간의 커밋 활동을 트래킹하고 있는데, `cpdax_admin` 프로젝트의 활동량이 늘어난 것을 수치로 확인하니 뿌듯함이 느껴집니다.

단순히 코드를 짜는 것을 넘어, 내가 어디에 시간을 쓰고 있고 어떤 프로젝트가 활발히 진행 중인지 데이터로 시각화하는 습관은 스스로의 생산성을 관리하는 데 큰 도움이 됩니다.

### 마치며: 엔지니어의 인사이트
오늘의 작업은 거창한 알고리즘 구현이나 시스템 설계는 아니었습니다. 하지만 **"잘못된 IP 수정"**과 **"루트 경로 리다이렉트"**라는 두 작업은 결국 시스템의 **안정성(Stability)**과 **사용성(Usability)**이라는 핵심 가치와 연결되어 있습니다.

기술적 깊이만큼이나 중요한 것은, 내가 만든 시스템이 실제 환경에서 어떻게 작동하고 사용되는지를 세심하게 살피는 태도라는 것을 다시 한번 느낀 하루였습니다.

내일은 더 깊이 있는 기술적 도전으로 돌아오겠습니다.
